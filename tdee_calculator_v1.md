# TDEE 계산기 구현 요청서 v1

## 목표

React + Vite + styled-components 기반으로 동작하는 TDEE 계산기 웹 서비스를 구현한다.

이 프로젝트는 1차 버전이며, 기능 완성 및 기본적인 사용자 경험 구현이 목적이다.

---

## 작업 방식

- 기존 Vite 기본 템플릿 UI는 사용하지 않는다.
- 기존 `src/App.jsx` 내용은 전부 제거하고 새로 작성한다.
- `src/main.jsx`는 기본 React 렌더 구조를 유지해도 된다.
- 스타일링은 `styled-components`를 사용한다.
- 컴포넌트 분리는 하지 않고, `src/App.jsx` 하나에서 구현한다.

---

## 수정 대상 파일

- `src/App.jsx` → 전체 새로 작성
- `src/main.jsx` → 필요 시 최소 수정
- `src/index.css` 또는 `src/App.css` → 사용하지 않아도 됨. 사용하지 않는다면 import 제거

---

## 서비스 설명

사용자의 기초대사량(BMR)과 활동량을 기반으로 하루 총 에너지 소비량(TDEE)을 계산하는 웹 서비스

---

## 입력값

### 필수 입력
- 성별: 남 / 여
- 나이: 숫자 입력
- 키: cm 단위 숫자 입력
- 몸무게: kg 단위 숫자 입력
- 활동량: 5단계 중 1개 선택

### 활동량 단계
- 1단계: 거의 운동하지 않음
- 2단계: 가벼운 활동 (주 1~3회)
- 3단계: 보통 활동 (주 3~5회)
- 4단계: 높은 활동 (주 6~7회)
- 5단계: 매우 높은 활동 (강도 높은 운동 또는 육체 노동)

### 선택 입력
- 체지방률
- 목표

단, 1차 버전에서는 선택 입력값은 화면에 포함하지 않아도 된다.  
핵심 기능은 필수 입력값 기반 TDEE 계산이다.

---

## 입력 검증

아래 검증을 구현한다.

- 모든 필수 입력값은 반드시 입력되어야 한다.
- 나이, 키, 몸무게는 숫자만 입력 가능해야 한다.
- 나이, 키, 몸무게는 0보다 큰 값이어야 한다.
- 활동량은 반드시 하나 선택되어야 한다.

검증 실패 시:
- 계산하지 않는다.
- 사용자에게 이해 가능한 문구로 오류 메시지를 보여준다.

예시:
- 성별을 선택해주세요.
- 나이를 입력해주세요.
- 키는 0보다 큰 숫자여야 합니다.
- 활동량을 선택해주세요.

---

## 계산 로직

### 1. BMR 계산식 (Mifflin-St Jeor)

#### 남자
BMR = (10 × 몸무게) + (6.25 × 키) - (5 × 나이) + 5

#### 여자
BMR = (10 × 몸무게) + (6.25 × 키) - (5 × 나이) - 161

---

### 2. 활동계수

- 1단계: 1.2
- 2단계: 1.375
- 3단계: 1.55
- 4단계: 1.725
- 5단계: 1.9

---

### 3. TDEE 계산식

TDEE = BMR × 활동계수

---

### 4. 활동대사량 계산식

활동대사량 = TDEE - BMR

---

### 5. 결과값 처리

- BMR, 활동대사량, TDEE는 모두 반올림하여 정수로 표시한다.

---

## 출력값

결과 영역에는 아래 항목을 보여준다.

- 기초대사량 (BMR)
- 활동대사량
- TDEE
- 해설형 설명 문구

### 해설형 설명 문구 예시
- 기초대사량은 약 1,500kcal입니다.
- 활동량을 반영한 하루 총 소모 칼로리는 약 2,100kcal입니다.
- 현재 활동 수준을 기준으로 계산된 결과입니다.

문구는 자연스러운 한국어로 출력한다.

---

## UI 구조

하나의 화면에서 입력과 결과를 모두 처리한다.

### 화면 구성 순서

1. 페이지 제목
2. 입력 영역
3. 계산하기 버튼
4. 결과 영역

---

## 입력 영역 상세

### 1. 성별
- 남 / 여 선택 방식
- 버튼형 또는 라디오형 가능
- 직관적으로 선택 가능해야 한다

### 2. 나이
- 숫자 입력 필드

### 3. 키
- 숫자 입력 필드
- 단위 텍스트 `cm` 표시 가능

### 4. 몸무게
- 숫자 입력 필드
- 단위 텍스트 `kg` 표시 가능

### 5. 활동량
- 라디오 버튼 방식
- 5개 선택지가 모두 화면에 보이도록 구현
- 각 선택지 아래에 설명 문구를 함께 표시

---

## 결과 영역 동작

- 초기 상태에서는 결과 영역을 숨긴다.
- 사용자가 `계산하기` 버튼을 클릭하면:
  1. 입력 검증 수행
  2. 계산 수행
  3. 결과 영역 표시
  4. 화면이 결과 영역으로 자동 스크롤 이동되도록 한다

즉, 버튼 클릭 후 결과 영역이 사용자 시야에 들어오도록 포커싱 처리한다.

---

## 숫자 애니메이션

결과값은 즉시 최종 숫자를 보여주지 않는다.

아래 방식으로 구현한다.

- BMR, 활동대사량, TDEE 모두
- 0부터 시작해서 최종 숫자까지 증가하는 카운트업 애니메이션 적용
- 과도하게 느리지 않게 구현
- 사용자가 보기 좋은 수준의 자연스러운 속도로 표시

---

## 디자인 방향

1차 버전이지만, 너무 투박하지 않게 구현한다.

### 디자인 요구사항
- 전체 화면은 깔끔하고 현대적인 계산기 UI 느낌
- 카드형 레이아웃 사용 가능
- 여백 충분히 사용
- 입력 필드와 결과 영역이 명확히 구분되어야 함
- 모바일/데스크탑 모두 무리 없이 보이도록 반응형 구성
- 색상은 과하지 않게, 기본적으로 깔끔한 톤 사용
- 버튼은 명확하게 클릭 가능해야 함
- 결과 영역은 시각적으로 강조

---

## 기술 요구사항

- React 함수형 컴포넌트로 구현
- 상태 관리는 `useState` 사용
- 필요 시 `useMemo`, `useEffect`, `useRef` 사용 가능
- styled-components 사용
- 외부 상태관리 라이브러리 사용 금지
- 불필요한 패키지 추가 금지

---

## 구현 결과 기준

아래 조건을 만족해야 완료로 본다.

1. 사용자가 성별, 나이, 키, 몸무게, 활동량을 입력할 수 있다.
2. 잘못된 입력 시 오류 문구가 보인다.
3. 정상 입력 시 BMR, 활동대사량, TDEE가 계산된다.
4. 결과 영역이 계산 후 표시된다.
5. 결과 영역으로 자동 스크롤 이동된다.
6. 숫자가 0부터 증가하는 애니메이션으로 표시된다.
7. 전체 UI가 기본 React 템플릿이 아니라 실제 서비스 형태로 보인다.
