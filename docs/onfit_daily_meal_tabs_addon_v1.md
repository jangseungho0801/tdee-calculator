# OnFit Daily Meal Page - Meal Structure Tabs Add-on Spec v1

## 문서 목적
이 문서는 **현재 이미 개발 반영된 daily-meal 페이지를 기준으로**,  
식사 구조 섹션을 **평일 / 주말 2개 탭 구조**로 확장하기 위한 코덱스 CLI 전달용 MD 파일이다.

중요:
- 이 문서는 daily-meal 페이지 전체를 새로 만드는 문서가 아니다.
- 이미 반영된 상단 타이틀/설명, 목표 설정 카드, 상태 요약 카드, 목표 검증/모달 로직은 유지한다.
- 이번 작업 범위는 **식사 구조 섹션을 평일/주말 탭 구조로 변경 및 정리**하는 것에 한정한다.

---

## 1. 전제
현재 daily-meal 페이지에는 아래가 이미 구현되어 있다고 본다.

- 페이지 URL: `daily-meal`
- 카드 밖 최상단 타이틀
  - `Step3 어디까지, 언제까지 갈지 정해보세요`
- 카드 밖 최상단 설명
  - `현재 체중을 기준으로 목표 체중과 기간을 설정해보세요`
- 공통 목표 설정 카드
  - 현재 목표 / 현재 체중 요약
  - 목표 체중 / 목표 기간
  - 유지일 때 목표 체중 입력칸 숨김
- 목표 검증 및 재확인 모달 로직
- 페이지 하단 `다음` CTA
- 식사 구조 섹션 기본 추가 버전

이번 작업은 위 내용을 유지하면서,  
**식사 구조 섹션을 평일/주말 탭 구조로 변경**하는 작업이다.

---

## 2. 작업 목표
daily-meal 페이지 안에서 사용자가 아래를 **평일 / 주말 각각 따로** 설정할 수 있게 만든다.

1. 식사 개수
2. 각 식사 이름
3. 각 식사 시간
4. 식사 순서
5. 식사 추가 / 삭제

즉, 목표 설정은 공통으로 두고,  
**식사 구조만 평일 / 주말로 나누어 설정**하게 만드는 것이 목적이다.

---

## 3. 구조 원칙
### 전체 페이지 구조
- 카드 밖 최상단: Step3 타이틀 + 설명
- 카드 1: 공통 목표 설정 카드
- 카드 2: 식사 구조 카드
- 페이지 하단: `다음` CTA

### 핵심 원칙
- **목표 설정은 공통**
- **식사 구조만 평일 / 주말 분리**

주의:
- 목표 체중 / 목표 기간을 평일/주말 탭 안에서 중복 입력받지 않는다
- 식사 구조만 탭으로 분리한다

---

## 4. 식사 구조 카드 구성

### 카드 형태
- 목표 설정 카드와 **별도의 카드**로 구성

### 카드 내부 상단
- 탭 2개 배치
  - `평일`
  - `주말`

### 기본 선택 탭
- `평일`

주의:
- 첫 진입 시 기본 선택은 항상 평일이어야 한다

---

## 5. 탭별 기본값

### 평일 탭 기본값
기본 식사 3개로 시작한다.

- `아침`
- `점심`
- `저녁`

### 주말 탭 기본값
기본 식사 2개로 시작한다.

- `아점`
- `저녁`

주의:
- 평일과 주말은 기본 식사 구조가 다르게 시작해야 한다
- 복사 기능은 넣지 않는다

---

## 6. 식사 개수 규칙
### 공통 규칙
- 최소 1개
- 최대 10개

주의:
- 사용자는 식사를 추가/삭제할 수 있어야 한다
- 단, 0개는 허용하지 않는다
- 10개를 초과해서 추가할 수 없게 한다

---

## 7. 섹션 타이틀
### 타이틀
`하루에 얼마나 드실 수 있으세요?`

### 설명
- 없음

주의:
- 식사 구조 카드는 탭 구조로 바꾸더라도 별도 설명 문구는 넣지 않는다
- 타이틀만으로 역할이 전달되게 한다

---

## 8. 식사 목록 구성
각 식사는 **이름 + 시간**으로 구성한다.

### 배치 방식
- **한 줄 배치**

### 비율
- 식사 이름: **7/10**
- 시간: **3/10**

예시:
- `아침                 08:00`
- `점심                 13:00`
- `저녁                 19:00`

주의:
- 이름과 시간을 두 줄로 쪼개지 않는다
- 탭 구조 안에서도 세로 길이를 과도하게 늘리지 않는 배치를 우선한다

---

## 9. 식사 이름 입력
### 입력 방식
- 자유 텍스트 입력

### 검증
- 빈값 불가

### 의도
기본값은 `아침 / 점심 / 저녁` 또는 `아점 / 저녁`이지만, 사용자가 아래처럼 변경할 수 있어야 한다.

예:
- `운동 전 식사`
- `오후 간식`
- `늦은 저녁`

주의:
- 완전 자유 입력은 허용하되, 식사 이름을 비워두는 것은 허용하지 않는다

---

## 10. 식사 시간 입력
### 입력 방식
- 각 식사마다 시간 1개 입력
- **타임피커** 사용

주의:
- 기존 HH:MM 직접 입력 필드가 아니라 타임피커 기반으로 구현
- 시작/종료 시간 범위 입력이 아니라, **시간 1개만 입력**한다

---

## 11. 식사 추가
### 버튼 문구
`식사 추가하기`

### 위치
- 식사 목록 바로 아래
- **좌측 정렬**

주의:
- 전체 폭 버튼으로 만들지 않는다
- 페이지 하단 `다음` CTA와 경쟁하지 않게 한다

---

## 12. 식사 삭제
### 방식
- 휴지통 아이콘 사용
- 텍스트 버튼은 사용하지 않는다

### 보조 처리
- 툴팁 또는 접근성 라벨 처리
- 터치 영역은 충분히 크게

예시 접근성 라벨:
- `아침 삭제`
- `점심 삭제`
- `아점 삭제`

주의:
- 반복 리스트에서 `삭제` 텍스트 버튼을 여러 개 노출하지 않는다

---

## 13. 식사 순서 변경
### 방식
- **위 버튼 / 아래 버튼** 사용

주의:
- 드래그 핸들 방식은 사용하지 않는다
- 각 식사 항목의 순서를 위/아래 버튼으로 바꿀 수 있어야 한다
- 순서 변경 시 항목 개수는 유지되어야 한다
- 이름/시간 값은 그대로 유지되어야 한다

---

## 14. 복사 기능
- **사용하지 않는다**

주의:
- 평일 식사 복사 / 주말 식사 복사 기능을 추가하지 않는다
- 평일/주말 식사 구조는 각각 직접 설정하는 방식으로 유지한다

---

## 15. CTA
### 유지 규칙
- 기존 daily-meal 페이지 하단의 `다음` CTA를 그대로 사용한다
- 식사 구조 카드용 별도 저장 버튼은 추가하지 않는다

즉,
- 공통 목표 설정
- 평일 식사 구조
- 주말 식사 구조
를 모두 설정한 뒤, 페이지 하단 `다음` 버튼 하나로 다음 단계로 넘어가게 한다.

---

## 16. 디자인 방향
OnFit 디자인 시스템 v1을 참고한다.

### 핵심 방향
- 화이트 기반 미니멀 UI
- 조용한 기본 위에 작은 포인트만 사용
- 식사 구조 카드는 기존 daily-meal 톤을 해치지 않게 추가
- 과한 장식, 과한 애니메이션 금지

### 구현 포인트
- 목표 설정 카드와 식사 구조 카드는 별도 카드로 분리
- 식사 구조 카드 안에서 평일/주말 탭이 명확히 구분되어야 한다
- 반복되는 식사 행(row)은 지나치게 무거운 카드형보다, 가볍고 정돈된 입력 행 구조를 우선한다

---

## 17. 금지 사항
아래 변경은 하지 않는다.

- 목표 설정을 평일/주말 탭 안으로 넣기
- 식사 구조를 별도 페이지로 분리하기
- 식사 구조 섹션에 별도 설명 문구 추가하기
- 식사 이름/시간을 두 줄 구조로 바꾸기
- 시간 입력을 자유 텍스트 HH:MM 필드로 되돌리기
- `삭제` 텍스트 버튼 반복 노출하기
- 순서 변경을 드래그 핸들 방식으로 되돌리기
- 복사 기능 추가하기
- 식사 구조 카드용 별도 저장 버튼 추가하기
- 기존 목표 설정 카드의 상단 타이틀/설명/모달 규칙을 임의로 수정하기

---

## 18. 구현 우선순위
1. 식사 구조 카드를 평일/주말 탭 구조로 변경
2. 기본 선택 탭을 평일로 고정
3. 평일 기본값 3개 반영
4. 주말 기본값 2개 반영
5. 식사 이름 + 시간 한 줄 배치 유지
6. 식사 이름 자유 입력 / 빈값 불가 반영
7. 시간 입력을 타임피커로 정리
8. `식사 추가하기` 버튼 반영
9. 휴지통 아이콘 삭제 기능 반영
10. 위/아래 버튼 순서 변경 반영
11. 기존 `다음` CTA와 연결 상태 점검

---

## 19. 최종 구현 기준
구현 결과는 아래 조건을 만족해야 한다.

- 식사 구조는 목표 설정과 별도의 카드로 구성되어야 한다
- 식사 구조 카드 안에는 `평일 / 주말` 2개 탭이 있어야 한다
- 기본 선택 탭은 `평일`이어야 한다
- 평일 기본 식사값은 `아침 / 점심 / 저녁`이어야 한다
- 주말 기본 식사값은 `아점 / 저녁`이어야 한다
- 식사 수는 탭별로 최소 1개, 최대 10개여야 한다
- 식사 이름은 자유 텍스트 입력 가능하되 빈값이면 안 된다
- 각 식사는 이름 + 시간을 한 줄에 배치해야 한다
- 비율은 식사 이름 7/10, 시간 3/10이어야 한다
- 시간 입력은 타임피커여야 한다
- `식사 추가하기` 버튼은 식사 목록 바로 아래 좌측 정렬이어야 한다
- 삭제는 휴지통 아이콘 + 보조 툴팁/접근성 처리여야 한다
- 순서 변경은 위/아래 버튼 방식이어야 한다
- 복사 기능은 없어야 한다
- 페이지 하단 CTA는 기존 `다음` 하나를 그대로 사용해야 한다
