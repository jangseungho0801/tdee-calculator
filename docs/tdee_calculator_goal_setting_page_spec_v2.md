# OnFit Goal Setting Page Spec v2

## 문서 목적
이 문서는 `OnFit 식단 관리` 서비스의 **목표 설정 페이지 구현 지시용 MD 파일**이다.  
코덱스 CLI는 이 문서를 기준으로 목표 설정 페이지를 구현한다.

---

## 1. 페이지 목적
목표 설정 페이지의 목적은 아래 3가지다.

1. 사용자가 목표 체중 또는 유지 목표를 설정하게 한다.
2. 목표 달성 기간을 입력하게 한다.
3. 비현실적인 목표는 한 번 더 확인하게 만든다.

이 페이지는 **숫자 입력 중심의 간단한 설정 페이지**다.  
설명 페이지처럼 길어지지 않도록 한다.

---

## 2. 페이지 성격
- 역할: 목표 설정
- 정의: 결과 확인 후, 사용자의 목표 체중과 목표 기간을 설정하는 페이지
- 톤앤매너: 코치형
- 말투: 직설적이지만 과하지 않게
- 목적: 사용자가 자신의 목표를 명확히 입력하게 만드는 것

---

## 3. 목표별 상단 질문 문구
상단 질문 문구는 결과 페이지에서 선택한 목표에 따라 다르게 노출한다.

### 다이어트
`언제까지 몇kg을 빼고 싶으세요?`

### 유지
`언제까지 지금 체중을 유지하고 싶으세요?`

### 벌크업
`언제까지 몇kg을 늘리고 싶으세요?`

주의:
- 상단 질문 문구는 고정 1개가 아니라 목표별 분기 문구로 처리한다.

---

## 4. 입력 구조

### 다이어트 / 벌크업
- 목표 체중
- 목표 기간

### 유지
- 목표 기간만 입력
- 목표 체중 입력칸은 아예 숨긴다

주의:
- 유지일 때 목표 체중 입력칸을 비활성화로 남기지 않는다.
- 유지는 기간만 입력하는 구조로 단순화한다.

---

## 5. 현재 체중 노출 방식

### 다이어트 / 벌크업
- 현재 체중은 `목표 체중` 라벨 옆에 정보형으로 노출한다.
- 예시:
  `목표 체중    현재 체중: 72.5kg`

### 유지
- 목표 체중 입력칸이 없으므로, 현재 체중은 상단 질문 아래에 정보형으로 노출한다.
- 예시:
  `현재 체중: 72.5kg`

주의:
- 현재 체중 값은 이전 정보 입력 페이지에서 입력한 현재 체중 값을 사용한다.
- 현재 체중은 사용자가 목표 체중을 비교하고 이해하기 위한 기준값이다.

---

## 6. 입력 방식

### 목표 체중
- 직접 숫자 입력만 사용

### 목표 기간
- 직접 숫자 입력 + 단위 선택
- 단위 선택 옵션:
  - `일`
  - `주`
  - `개월`

### 입력 순서
- 목표 체중 → 목표 기간

### 입력 UI 배치
- 목표 기간은 `숫자 입력창 + 단위 선택`을 한 줄에 배치

### 입력 라벨
- `목표 체중`
- `목표 기간`

---

## 7. CTA
### 버튼 라벨
`다음`

주의:
- CTA는 1개만 사용한다.
- CTA는 페이지 하단에 명확하게 배치한다.

---

## 8. 검증 기준

### 8.1 목표 체중 검증
이전 정보 입력 페이지에서 입력한 `현재 체중`을 기준값으로 사용한다.

#### 다이어트
- 목표 체중은 현재 체중보다 낮아야 한다

#### 벌크업
- 목표 체중은 현재 체중보다 높아야 한다

#### 유지
- 목표 체중 입력 없음

---

### 8.2 목표 기간 검증
다이어트 / 유지 / 벌크업 모두 목표 기간 최대는 **3개월**이다.

#### 기간 초과 안내 문구
- 다이어트 / 벌크업:
  `우선 최대 3개월 동안 식단을 진행하고 다시 목표를 설정하세요`
- 유지:
  `우선 최대 3개월 동안 유지 식단을 진행하고 다시 목표를 설정하세요`

---

### 8.3 변화량 대비 검증
다이어트 / 벌크업은 목표 체중 변화량과 목표 기간을 함께 계산해서 검증한다.

#### 다이어트
- 계산 결과 `주당 500g 초과 감량`이면 안내 필요

안내 문구:
`너무 급격한 감량은 건강에 좋지 않아요, 한번만 다시 생각해보세요`

#### 벌크업
- 계산 결과 `주당 500g 초과 벌크업`이면 안내 필요

안내 문구:
`너무 급격한 벌크업은 살크업 가능성이 높아요, 한번만 다시 생각해보세요`

#### 유지
- 변화량 검증 없음
- 기간만 검증

---

## 9. 단위 환산 기준
내부 계산 기준은 아래로 고정한다.

- 1일 = 1일
- 1주 = 7일
- 1개월 = 30일

주의:
- 일 / 주 / 개월 입력은 모두 위 기준으로 일수 환산 후 계산한다.

---

## 10. 안내 방식: 모달

### 트리거
- 사용자가 `다음` 버튼을 클릭했을 때
- 검증 조건에 걸리면 모달을 노출한다

### 모달 제목
`한번 더 확인해보세요`

### 모달 버튼
- `수정하기`
- `그대로 진행하기`

### 동작
- `수정하기`: 모달 닫기, 현재 페이지에서 입력 수정
- `그대로 진행하기`: 다음 단계로 이동

주의:
- 브라우저 기본 alert를 사용하지 않는다
- 선택 가능한 모달 UI로 구현한다

---

## 11. 복수 조건 충족 시 처리
조건이 여러 개 동시에 충족되면,  
해당 안내 문구를 **하나의 모달 안에 모두 노출**한다.

### 본문 형식
- 리스트형

### 예시
다이어트에서
- 기간 6개월
- 주당 1kg 감량 수준

이면 모달 본문은 리스트로 아래처럼 노출한다.

- `우선 최대 3개월 동안 식단을 진행하고 다시 목표를 설정하세요`
- `너무 급격한 감량은 건강에 좋지 않아요, 한번만 다시 생각해보세요`

---

## 12. 모달 재노출 규칙
- 사용자가 `그대로 진행하기`를 눌렀더라도,
- 같은 값으로 다시 `다음` 버튼을 누르면
- 조건에 걸리는 경우 모달을 다시 띄운다

주의:
- “한 번 봤으니 다음부터는 무시” 같은 예외 저장 로직은 두지 않는다.
- 매번 현재 입력값 기준으로 다시 검증한다.

---

## 13. 디자인 방향
온핏 디자인 시스템 v1의 방향을 참고한다.

### 핵심 방향
- 화이트 기반 미니멀 UI
- 조용한 기본 위에 작은 포인트만 사용
- Blue 계열 Primary CTA 사용
- Orange는 상시 기본 상태가 아니라 강조 순간에만 사용
- 과한 장식, 과한 애니메이션 금지

### 컴포넌트 방향
- 숫자 입력은 단순하고 읽기 쉬워야 한다
- 단위 선택은 직관적이어야 한다
- 모달은 현재 페이지 톤을 해치지 않게 깔끔하게 구성한다

---

## 14. 금지 사항
아래 변경은 하지 않는다.

- 유지일 때 목표 체중 입력칸을 남겨두기
- 목표 기간 입력을 슬라이더로 바꾸기
- 목표 체중 입력에 증감 버튼 추가하기
- 입력창 아래 에러 문구처럼 빨간 경고를 상시 노출하기
- alert 사용
- 모달 버튼을 `취소 / 확인`처럼 일반적인 문구로 바꾸기

---

## 15. 구현 의도
이 페이지의 핵심은 **사용자가 목표를 입력하게 하되, 너무 무리한 목표는 한 번 더 생각하게 만드는 것**이다.

즉,
- 무조건 막지 않는다
- 그렇다고 그냥 통과시키지도 않는다
- 코치처럼 한 번 더 생각하게 만든다

결과 페이지에서 방향을 선택했다면,  
목표 설정 페이지에서는 그 방향을 현실적인 목표로 구체화해야 한다.

---

## 16. 구현 우선순위
목표 설정 페이지 구현 시 우선순위는 아래 순서로 둔다.

1. 목표별 상단 질문 문구 분기
2. 유지일 때 목표 체중 입력칸 숨김 처리
3. 현재 체중 노출 규칙 반영
4. 목표 체중 / 목표 기간 입력 UI 구성
5. 목표 기간 단위 선택 한 줄 배치
6. 현재 체중 기준 검증 반영
7. 최대 3개월 검증 반영
8. 주당 500g 초과 변화량 검증 반영
9. 모달 구조 구현
10. 복수 조건 리스트형 본문 처리
11. `수정하기 / 그대로 진행하기` 버튼 동작 구현

---

## 17. 최종 구현 기준
구현 결과는 아래 조건을 만족해야 한다.

- 다이어트 / 유지 / 벌크업 선택값에 따라 상단 질문 문구가 달라져야 한다
- 다이어트 / 벌크업은 목표 체중 + 목표 기간을 입력받아야 한다
- 유지는 목표 기간만 입력받고, 목표 체중 입력칸은 보이지 않아야 한다
- 다이어트 / 벌크업에서는 현재 체중이 목표 체중 라벨 옆에 정보형으로 보여야 한다
- 유지에서는 현재 체중이 상단 질문 아래에 정보형으로 보여야 한다
- 현재 체중은 이전 정보 입력 페이지의 값을 사용해야 한다
- 목표 체중은 직접 숫자 입력만 사용해야 한다
- 목표 기간은 숫자 입력 + 단위 선택 구조여야 한다
- 목표 기간 입력은 한 줄 배치여야 한다
- 입력 라벨은 `목표 체중`, `목표 기간`이어야 한다
- CTA 버튼 라벨은 `다음`이어야 한다
- 기간은 최대 3개월까지만 허용해야 한다
- 다이어트 / 벌크업은 주당 500g 초과 변화 시 재확인 모달을 띄워야 한다
- 유지 기간 초과 시 유지 전용 문구를 사용해야 한다
- 모달 제목은 `한번 더 확인해보세요`여야 한다
- 모달 버튼은 `수정하기 / 그대로 진행하기`여야 한다
- 조건이 여러 개면 모달 본문에 리스트형으로 모두 보여야 한다
- 같은 값으로 다시 `다음`을 눌러도 조건에 걸리면 모달이 다시 떠야 한다
