# OnFit Flow Update Spec v1

## 문서 목적
이 문서는 `OnFit 식단 관리` 서비스의 **코덱스 CLI 전달용 통합 구현 문서**이다.  
코덱스 CLI는 이 문서를 기준으로 아래 3개 페이지와 공통 규칙을 한 번에 반영한다.

- 정보 입력 페이지
- 결과 페이지
- 목표 설정 페이지

이 문서는 개별 페이지 문서가 아니라, **플로우 단위 통합 문서**다.

---

## 1. 공통 방향

### 1.1 서비스 흐름
현재 단계 기준 플로우는 아래와 같다.

1. 인트로 페이지
2. 정보 입력 페이지
3. 결과 페이지
4. 목표 설정 페이지

---

### 1.2 공통 톤앤매너
- 코치형
- 직설적이지만 과하지 않게
- 설명은 짧게
- 숫자 입력과 선택 경험을 방해하지 않게
- 화이트 기반 미니멀 UI
- OnFit 디자인 시스템 v1 참고

---

### 1.3 공통 유효성 규칙
사용자가 필수 입력 항목을 채우지 않은 상태에서 `다음`을 누르면:

- 첫 번째 미입력 항목으로 포커싱
- 여러 개가 비어 있으면 **위에서부터 첫 번째 미입력 항목**을 우선 포커싱
- 가능하면 해당 위치로 스크롤도 함께 이동

주의:
- 여러 개 항목을 동시에 흔들거나 포커싱하지 않는다
- 가장 먼저 수정해야 할 항목 하나에 집중시키는 방식으로 구현한다

---

### 1.4 공통 URL 변경
- 정보 입력 페이지: `input` → `basic-info`
- 결과 페이지: `result` → `daily-calories`

주의:
- URL은 소문자 + 하이픈 기준으로 유지한다

---

## 2. 정보 입력 페이지

### 2.1 페이지 역할
- 계산에 필요한 정보를 빠르게 입력하게 한다
- 헷갈릴 수 있는 항목만 짧게 안내한다
- 입력 완료 후 결과 페이지로 자연스럽게 연결한다

---

### 2.2 상단 구조
상단의 `INPUT` 라벨은 삭제한다.

#### 타이틀
`닭가슴살부터 먹지 말고, 기준부터 확인하세요`

#### 설명
`기본 정보와 활동량을 입력하면 내 몸에 맞는 섭취 기준을 계산할 수 있어요`

---

### 2.3 전체 구조
정보 입력 페이지는 **2개 섹션 구조**로 구성한다.

#### 섹션 1
`기본 정보`

#### 섹션 2
`활동 및 추가 정보`

---

### 2.4 기본 정보 섹션
#### 입력 순서
1. 성별
2. 나이
3. 키
4. 현재 체중

#### 성별 입력 방식
- 가로 2분할 선택형
- 항목:
  - `남성`
  - `여성`

#### 나이 입력 방식
- 직접 숫자 입력

#### 키 입력 방식
- 직접 숫자 입력

#### 현재 체중 입력 방식
- 직접 숫자 입력

#### 배치
- `나이`: 한 줄 단독 배치
- `키 / 현재 체중`: 한 줄 2열 배치

---

### 2.5 활동 및 추가 정보 섹션
#### 항목명
`평소 활동량`

#### 입력 방식
- 세로 카드형 리스트 유지
- 드롭다운으로 바꾸지 않는다
- 가로 탭형으로 바꾸지 않는다

#### 기본 선택값
- `앉아 있는 시간과 움직이는 시간이 비슷해요`

주의:
- 기본 선택 상태가 UI에서 명확히 보여야 한다

#### 선택지 개수
- 5단계 유지

#### 선택지 라벨
1. `하루 대부분 앉아서 보내요`
2. `앉아 있는 시간이 많지만 조금씩 움직여요`
3. `앉아 있는 시간과 움직이는 시간이 비슷해요`
4. `움직이는 시간이 많은 편이에요`
5. `하루 대부분 몸을 많이 쓰는 편이에요`

주의:
- 추상어(낮음/보통/높음)로 바꾸지 않는다
- 운동 횟수 중심 문구로 바꾸지 않는다
- 라벨 자체가 생활 문장 역할을 해야 한다

---

### 2.6 체지방률
#### 노출 방식
- `활동 및 추가 정보` 섹션 안에 기본 노출

#### 라벨
`체지방률 (선택)`

#### 보조 문구
`모르면 비워두셔도 괜찮아요`

#### 입력 방식
- 직접 숫자 입력

---

### 2.7 CTA
#### 버튼 문구
`내 기준 보러가기`

#### CTA 위 보조 문구
- 없음

---

## 3. 결과 페이지

### 3.1 페이지 역할
- 계산 결과를 명확하게 보여준다
- 다이어트 / 유지 / 벌크업 기준을 비교하게 만든다
- 결과를 보고 끝나는 것이 아니라, 다음 행동으로 이어지게 만든다

---

### 3.2 상단 구조
#### 페이지 타이틀
`Step2 하루 권장 칼로리를 확인하고, 하루 식단을 구성하세요`

주의:
- 정보 입력 페이지처럼 결과 페이지에도 Step 타이틀이 있어야 한다

---

### 3.3 결과 상단 숫자 요약 영역
#### 구성
- `기초대사량`
- `유지 칼로리`

#### 배치
- 같은 크기의 2개 카드로 나란히 표시

주의:
- `BMR`, `TDEE` 영문 라벨 그대로 쓰지 않는다
- `기초대사량 / 유지 칼로리` 한글 기준으로 유지한다

---

### 3.4 목표 선택 버튼 영역
#### 버튼 구조
- 가로 3분할 배치

#### 버튼 라벨
- `다이어트`
- `유지`
- `벌크업`

#### 동작
- 버튼 클릭 시 아래 상세 결과가 해당 목표 기준으로 바뀌어야 한다

주의:
- 기존에 있던 아래 문구 섹션은 삭제한다.
  - `덜 먹는 게 아니라, 내 기준보다 맞게 줄여야 해요`
  - `지금 몸 상태를 유지하고 싶다면 이 기준이 필요해요`
  - `무작정 많이 먹지 말고, 늘리는 기준부터 확인하세요`

---

### 3.5 선택 결과 상세 영역
구조는 아래 순서로 구성한다.

#### 상단
숫자 카드 영역

#### 하단
설명 문구 영역

즉:
`숫자 카드 위 / 설명 문구 아래`

---

### 3.6 숫자 카드 영역
#### 포함 정보
- 권장 칼로리 1개
- 탄수화물 g / %
- 단백질 g / %
- 지방 g / %

#### 배치
- 권장 칼로리 카드는 상단에 크게 노출
- 탄수화물 / 단백질 / 지방 카드는 그 아래 배치

#### 반응형 기준
- 모바일: 탄 / 단 / 지 카드 세로 1열
- 화면이 넓어지면: 탄 / 단 / 지 카드 가로 3분할

#### 매크로 카드 정보 순서
- `라벨 → g → %`

주의:
- 모바일에서 무조건 3분할 고정하지 않는다
- 숫자 가독성이 우선이다

---

### 3.7 설명 문구 영역
숫자 카드 아래에는 목표별 설명 문구를 노출한다.

#### 구조
- 설명 타이틀 1줄
- 설명 부가 1줄

#### 다이어트
- 설명 타이틀: `다이어트를 위한 하루 권장 칼로리에요`
- 설명 부가: `처음부터 너무 많이 줄이면 오래 유지하기 어려워요`

#### 유지
- 설명 타이틀: `현재 체중을 유지하기 위한 하루 권장 칼로리에요`
- 설명 부가: `활동량이 달라지면 이 기준도 달라질 수 있어요`

#### 벌크업
- 설명 타이틀: `벌크업을 위한 하루 권장 칼로리에요`
- 설명 부가: `무작정 많이 먹기보다 천천히 늘리는 게 더 좋아요`

---

### 3.8 하단 CTA
#### 구성
- `입력 다시하기` 버튼 1개 유지

---

## 4. 목표 설정 페이지

### 4.1 페이지 역할
- 사용자가 목표 체중 또는 유지 목표를 설정하게 한다
- 목표 달성 기간을 입력하게 한다
- 비현실적인 목표는 한 번 더 확인하게 만든다

이 페이지는 **숫자 입력 중심의 간단한 설정 페이지**다.

---

### 4.2 상단 구조
#### 페이지 타이틀
`Step3 어디까지, 언제까지 갈지 정해보세요`

#### 페이지 설명
`현재 체중을 기준으로 목표 체중과 기간을 설정해보세요`

주의:
- 목표 설정 페이지에도 정보 입력 페이지처럼 타이틀과 설명이 있어야 한다

---

### 4.3 목표별 상단 질문 문구
상단 질문 문구는 결과 페이지에서 선택한 목표에 따라 다르게 노출한다.

#### 다이어트
`언제까지 몇kg을 빼고 싶으세요?`

#### 유지
`언제까지 지금 체중을 유지하고 싶으세요?`

#### 벌크업
`언제까지 몇kg을 늘리고 싶으세요?`

---

### 4.4 입력 구조
#### 다이어트 / 벌크업
- 목표 체중
- 목표 기간

#### 유지
- 목표 기간만 입력
- 목표 체중 입력칸은 아예 숨긴다

---

### 4.5 현재 체중 노출 방식
#### 다이어트 / 벌크업
- 현재 체중은 `목표 체중` 라벨 옆에 정보형으로 노출
- 예시:
  `목표 체중    현재 체중: 72.5kg`

#### 유지
- 목표 체중 입력칸이 없으므로, 현재 체중은 상단 질문 아래에 정보형으로 노출
- 예시:
  `현재 체중: 72.5kg`

주의:
- 현재 체중 값은 이전 정보 입력 페이지에서 입력한 현재 체중 값을 사용한다

---

### 4.6 입력 방식
#### 목표 체중
- 직접 숫자 입력만 사용

#### 목표 기간
- 직접 숫자 입력 + 단위 선택
- 단위 선택 옵션:
  - `일`
  - `주`
  - `개월`

#### 입력 순서
- 목표 체중 → 목표 기간

#### 입력 UI 배치
- 목표 기간은 `숫자 입력창 + 단위 선택`을 한 줄에 배치

#### 입력 라벨
- `목표 체중`
- `목표 기간`

---

### 4.7 CTA
#### 버튼 라벨
`다음`

---

### 4.8 검증 기준
#### 목표 체중 검증
이전 정보 입력 페이지에서 입력한 `현재 체중`을 기준값으로 사용한다.

##### 다이어트
- 목표 체중은 현재 체중보다 낮아야 한다

##### 벌크업
- 목표 체중은 현재 체중보다 높아야 한다

##### 유지
- 목표 체중 입력 없음

---

#### 목표 기간 검증
다이어트 / 유지 / 벌크업 모두 목표 기간 최대는 **3개월**

##### 기간 초과 안내 문구
- 다이어트 / 벌크업:
  `우선 최대 3개월 동안 식단을 진행하고 다시 목표를 설정하세요`
- 유지:
  `우선 최대 3개월 동안 유지 식단을 진행하고 다시 목표를 설정하세요`

---

#### 변화량 대비 검증
다이어트 / 벌크업은 목표 체중 변화량과 목표 기간을 함께 계산해서 검증한다.

##### 다이어트
- 계산 결과 `주당 500g 초과 감량`이면 안내 필요
- 안내 문구:
  `너무 급격한 감량은 건강에 좋지 않아요, 한번만 다시 생각해보세요`

##### 벌크업
- 계산 결과 `주당 500g 초과 벌크업`이면 안내 필요
- 안내 문구:
  `너무 급격한 벌크업은 살크업 가능성이 높아요, 한번만 다시 생각해보세요`

##### 유지
- 변화량 검증 없음
- 기간만 검증

---

### 4.9 단위 환산 기준
내부 계산 기준은 아래로 고정한다.

- 1일 = 1일
- 1주 = 7일
- 1개월 = 30일

---

### 4.10 안내 방식: 모달
#### 트리거
- 사용자가 `다음` 버튼을 클릭했을 때
- 검증 조건에 걸리면 모달을 노출한다

#### 모달 제목
`한번 더 확인해보세요`

#### 모달 버튼
- `수정하기`
- `그대로 진행하기`

#### 동작
- `수정하기`: 모달 닫기, 현재 페이지에서 입력 수정
- `그대로 진행하기`: 다음 단계로 이동

주의:
- 브라우저 기본 alert 사용 금지
- 선택 가능한 모달 UI로 구현

---

### 4.11 복수 조건 충족 시 처리
조건이 여러 개 동시에 충족되면,  
해당 안내 문구를 **하나의 모달 안에 모두 노출**한다.

#### 본문 형식
- 리스트형

#### 예시
다이어트에서
- 기간 6개월
- 주당 1kg 감량 수준

이면 모달 본문은 아래처럼 리스트로 노출한다.

- `우선 최대 3개월 동안 식단을 진행하고 다시 목표를 설정하세요`
- `너무 급격한 감량은 건강에 좋지 않아요, 한번만 다시 생각해보세요`

---

### 4.12 모달 재노출 규칙
- 사용자가 `그대로 진행하기`를 눌렀더라도
- 같은 값으로 다시 `다음` 버튼을 누르면
- 조건에 걸리는 경우 모달을 다시 띄운다

---

## 5. 디자인 방향
온핏 디자인 시스템 v1의 방향을 참고한다.

### 핵심 방향
- 화이트 기반 미니멀 UI
- 조용한 기본 위에 작은 포인트만 사용
- Blue 계열 Primary CTA 사용
- Orange는 상시 기본 상태가 아니라 강조 순간에만 사용
- 과한 장식, 과한 애니메이션 금지

---

## 6. 금지 사항
아래 변경은 하지 않는다.

- 유지일 때 목표 체중 입력칸을 남겨두기
- 목표 기간 입력을 슬라이더로 바꾸기
- 목표 체중 입력에 증감 버튼 추가하기
- 입력창 아래 에러 문구처럼 빨간 경고를 상시 노출하기
- alert 사용
- 모달 버튼을 `취소 / 확인`처럼 일반적인 문구로 바꾸기

## 7. 구현 우선순위
1. 공통 미입력 포커싱 규칙 반영
2. 정보 입력 페이지 URL 변경 및 기본 선택값 반영
3. 결과 페이지 URL 변경 및 Step 타이틀 반영
4. 결과 페이지 삭제 대상 문구 섹션 제거
5. 결과 페이지 설명 문구 영역 반영
6. 목표 설정 페이지 타이틀 / 설명 반영
7. 목표 설정 페이지 현재 체중 노출 규칙 반영
8. 목표 설정 페이지 검증 로직 반영
9. 목표 설정 페이지 모달 구조 구현
10. 모달 복수 조건 리스트형 처리 반영

## 8. 최종 구현 기준
구현 결과는 아래 조건을 만족해야 한다.

- 필수 입력 누락 시 첫 번째 미입력 항목에 포커싱되어야 한다
- 여러 개 미입력 시 위에서부터 첫 번째 미입력 항목만 우선 포커싱되어야 한다
- 정보 입력 페이지 URL은 `basic-info`여야 한다
- 정보 입력 페이지 평소 활동량 기본 선택값은 `앉아 있는 시간과 움직이는 시간이 비슷해요`여야 한다
- 결과 페이지 URL은 `daily-calories`여야 한다
- 결과 페이지 타이틀은 `Step2 하루 권장 칼로리를 확인하고, 하루 식단을 구성하세요`여야 한다
- 결과 페이지 상세 영역에는 설명 타이틀 / 설명 부가 문구가 목표별로 보여야 한다
- 삭제 대상 3개 문구 섹션은 결과 페이지에서 보이지 않아야 한다
- 목표 설정 페이지 타이틀은 `Step3 어디까지, 언제까지 갈지 정해보세요`여야 한다
- 목표 설정 페이지 설명은 `현재 체중을 기준으로 목표 체중과 기간을 설정해보세요`여야 한다
- 다이어트 / 벌크업에서는 현재 체중이 목표 체중 라벨 옆에 정보형으로 보여야 한다
- 유지에서는 현재 체중이 상단 질문 아래에 정보형으로 보여야 한다
- 유지에서는 목표 체중 입력칸이 보이지 않아야 한다
- 기간은 최대 3개월까지만 허용해야 한다
- 다이어트 / 벌크업은 주당 500g 초과 변화 시 재확인 모달을 띄워야 한다
- 유지 기간 초과 시 유지 전용 문구를 사용해야 한다
- 모달 제목은 `한번 더 확인해보세요`여야 한다
- 모달 버튼은 `수정하기 / 그대로 진행하기`여야 한다
- 조건이 여러 개면 모달 본문에 리스트형으로 모두 보여야 한다
- 같은 값으로 다시 `다음`을 눌러도 조건에 걸리면 모달이 다시 떠야 한다
