# tdee_calculator daily-meal + daily-macro Update Spec v1

## 문서 목적
이 문서는 `tdee_calculator` 프로젝트에서 아래 2개 페이지의 최신 수정 사항을 반영하기 위한 코덱스 CLI 전달용 MD 파일이다.

- daily-meal 페이지
- daily-macro 페이지

중요:
- 본 문서는 현재 개발 반영 상태를 기준으로 한 **수정/업데이트 문서**다.
- 기존 구조를 전부 새로 만드는 문서가 아니다.
- 이번 문서의 핵심은
  1. daily-meal에서 식사 구조 입력 방식을 수정하고
  2. daily-macro에서 한 끼 기준 확인 페이지를 정리하는 것이다.

---

# 1. daily-meal 페이지 수정 사항

## 1.1 페이지 역할
daily-meal 페이지의 역할은 아래와 같다.

1. 공통 목표 설정
2. 평일 / 주말 식사 구조 설정
3. 각 식사의 성격 선택
   - 어떤 영양소 위주로 먹을지
   - 양을 얼마나 먹을지

주의:
- 식사 시간 입력은 더 이상 받지 않는다
- 사용자는 식사 이름, 식사 순서, 식사 성격만 설정한다

---

## 1.2 식사 목록 입력 구조 변경
기존 식사 목록 행에서 **시간 입력을 삭제**한다.

### 변경 전
- 식사 이름
- 시간 입력
- 휴지통 버튼

### 변경 후
- 식사 이름 입력박스
- 드롭다운 1
- 드롭다운 2
- 휴지통 버튼

즉, 입력박스와 휴지통 버튼 사이에 드롭다운 2개를 추가한다.

---

## 1.3 드롭다운 1 규칙
### 목적
각 식사에서 **어떤 영양소 위주로 섭취할지** 선택한다.

### 선택지
- `탄수화물 중심`
- `단백질 중심`
- `지방 중심`

### 기본값
- `탄수화물 중심`

주의:
- 이 값은 daily-macro 계산에 반영되어야 한다
- 단순 저장용 값이 아니다

---

## 1.4 드롭다운 2 규칙
### 목적
각 식사에서 **양을 얼마나 먹을지** 선택한다.

### 선택지
- `적게`
- `보통`
- `많이`

### 기본값
- `보통`

주의:
- 이 값은 daily-macro 계산에 반영되어야 한다
- 단순 저장용 값이 아니다

---

## 1.5 드롭다운 선택값의 계산 반영 규칙
daily-meal에서 사용자가 고른 드롭다운 값은 daily-macro 계산에 반영한다.

### 양 선택 반영 방식
**고정 가중치 방식**
- `적게` = 0.8
- `보통` = 1.0
- `많이` = 1.2

### 의미
같은 탭(평일/주말) 안에서 각 식사의 양 선택값에 따라 가중치를 적용한 뒤,
전체 합 기준으로 다시 정규화해서 끼니별 kcal 비중을 계산한다.

즉:
- 적게 고른 끼니는 더 작게
- 많이 고른 끼니는 더 크게
- 최종적으로는 탭 내 총합이 맞도록 계산

---

### 영양소 중심 반영 방식
**해당 영양소를 강하게 상향(20%)**

### 의미
각 끼니의 총 kcal가 정해진 뒤,
그 끼니 안에서 선택한 중심 영양소 비중을 **기본 비율 대비 +20% 수준으로 상향**하고,
나머지 2개 영양소는 총합이 맞도록 자동 조정한다.

예:
- 탄수화물 중심 → 탄수화물 비율 상향
- 단백질 중심 → 단백질 비율 상향
- 지방 중심 → 지방 비율 상향

---

## 1.6 식사 구조 카드 문구 변경
### 변경 전
`하루에 얼마나 드실 수 있으세요?`

### 변경 후
`하루 식사는 어떻게 하고 싶으세요?`

---

## 1.7 CTA 문구 변경
### 다음 버튼
- 변경 전: `다음`
- 변경 후: `하루 식단 확인하기`

### 뒤로가기 버튼
- 변경 전: `뒤로가기`
- 변경 후: `하루 권장 칼로리 확인하기`

---

## 1.8 버튼 위치/배치 변경
### 변경 사항
- 버튼은 **카드 밖**에 있어야 한다
- 버튼은 **하단에 고정**되어야 한다
- `다음` 버튼과 `뒤로가기` 버튼 간격은 **daily-calories 페이지와 동일**하게 맞춘다

주의:
- 버튼 영역은 스크롤을 따라다니는 구조가 아니라, 하단 고정 구조로 간다
- 카드 내부 CTA 구조로 두지 않는다

---

## 1.9 식사 구조 카드 유지 사항
아래 규칙은 유지한다.

- 평일 / 주말 탭 구조 유지
- 기본 선택 탭은 `평일`
- 평일 기본값: `아침 / 점심 / 저녁`
- 주말 기본값: `아점 / 저녁`
- 식사 수는 최소 1개, 최대 10개
- 식사 이름은 자유 텍스트 입력 가능, 빈값 불가
- 식사 추가 버튼 문구: `식사 추가하기`
- 삭제는 휴지통 아이콘 + 보조 툴팁/접근성 처리
- 순서 변경은 위/아래 버튼 방식

---

# 2. daily-macro 페이지 구현 기준

## 2.1 페이지 역할
daily-macro 페이지의 역할은 아래와 같다.

1. 이전 페이지에서 설정한 목표를 달성하기 위해, 사용자가 한 끼에 얼마나 먹어야 하는지 보여준다
2. daily-meal에서 설정한 평일/주말 식사 구조를 기준으로 끼니별 기준을 계산해 보여준다
3. 사용자가 이 페이지를 보고
   - `아, 한 끼에 이만큼 먹어야 하는구나`
   를 이해하게 만든다
4. 그 다음 페이지에서
   - `그럼 어떤 걸 먹어야 하지?`
   로 자연스럽게 이어지게 한다

즉, 이 페이지는 **한 끼 기준 확인 페이지**다.

주의:
- 사용자가 여기서 kcal, 탄단지, 비중을 직접 수정하게 하지 않는다
- 확인 중심 페이지로 유지한다

---

## 2.2 URL
- `daily-macro`

---

## 2.3 상단 구조
### 페이지 타이틀
`Step4 한번에 얼마나 먹어야 하는지 확인하세요`

### 페이지 설명
`평일과 주말 식사 구조에 맞춰 한 끼 기준을 계산했어요`

---

## 2.4 탭 구조
### 탭 구성
- `평일`
- `주말`

### 기본 선택 탭
- `평일`

### 데이터 기준
- 각 탭은 daily-meal에서 입력한 식사 구조를 그대로 가져온다
- 식사 이름만 사용하고, 식사 시간은 더 이상 사용하지 않는다

예:
- 평일: 아침 / 점심 / 저녁
- 주말: 아점 / 저녁

---

## 2.5 계산 기준
### 기본 원칙
- 균등 분배를 기본으로 하되,
- daily-meal에서 선택한 `적게 / 보통 / 많이` 가중치를 끼니별 kcal 비중 계산에 반영한다
- daily-meal에서 선택한 `탄수화물 중심 / 단백질 중심 / 지방 중심` 값을 탄단지 비율 조정에 반영한다

즉, 이제는 단순히 식사 개수만으로 균등 분배하지 않는다.
**사용자가 daily-meal에서 선택한 식사 의도**가 실제 계산 결과에 반영되어야 한다.

---

## 2.6 끼니 카드 구조
각 끼니는 **카드형**으로 표시한다.

### 카드 안 정보 순서
1. 식사 이름
2. kcal
3. 탄수화물 / 단백질 / 지방

### 카드 표현 원칙
- 식사 이름은 카드 상단
- kcal는 카드 안에서 **가장 크게 강조**
- 탄수화물 / 단백질 / 지방은 카드 하단 **가로 3분할**

---

## 2.7 카드 안 선택값 표시
daily-macro 카드 안에는 사용자가 daily-meal에서 고른 아래 2개 선택값도 같이 보여줘야 한다.

- 영양소 중심
- 양

### 위치
- 식사 이름과 **같은 열의 우측 끝**

### 형태
- **배지 형태**

### 예시
- `아침                           [단백질 중심] [많이]`

주의:
- 사용자는 이 배지를 보고 “왜 이 끼니 결과가 이렇게 계산됐는지” 이해할 수 있어야 한다

---

## 2.8 카드 정렬 방식
- **카드 1열**

주의:
- 현재 2열로 보이는 구조는 사용하지 않는다
- 각 끼니 카드를 위에서 아래로 한 장씩 세로로 보여준다

---

## 2.9 카드 안 정보 표현 방식

### kcal
- 카드 안에서 가장 크게 강조

예:
- `600 kcal`

### 탄수화물 / 단백질 / 지방
- 가로 3분할
- 풀네임 기준으로 표시

예:
- `탄수화물 60g`
- `단백질 40g`
- `지방 20g`

주의:
- `탄 / 단 / 지` 축약형 대신 풀네임 기준으로 표시한다

---

## 2.10 카드 하단 보조 문구
- 없음

주의:
- 카드마다 별도 설명 문구를 반복해서 넣지 않는다
- 이 페이지는 한 끼 기준 확인에 집중한다

---

## 2.11 CTA
### 버튼 문구
- `다음`
- `뒤로가기`

### 배치
- 버튼은 **카드 밖**에 있어야 한다
- 버튼은 **하단에 고정**되어야 한다
- 위: `다음`
- 아래: `뒤로가기`

주의:
- 하단 고정 버튼 영역은 basic-info / daily-meal / daily-macro 모두 **공통 스타일**로 맞춘다

# 6. 금지 사항
아래 변경은 하지 않는다.

- daily-meal에서 식사 시간 입력을 다시 추가하기
- daily-macro에서 식사 이름/시간을 다시 입력받기
- daily-macro를 수정형 페이지로 바꾸기
- daily-macro 카드 2열 유지하기
- 하단 CTA 영역을 페이지별 제각각 다르게 구성하기
- daily-meal 드롭다운 값을 단순 저장만 하고 계산에는 반영하지 않기

---

# 7. 구현 우선순위
1. basic-info CTA 문구/컬러/하단 고정 반영
2. daily-calories 버튼 라벨 변경
3. daily-meal 식사 시간 입력 제거
4. daily-meal 드롭다운 2개 추가 및 기본값 반영
5. daily-meal CTA 문구/위치/하단 고정 반영
6. daily-macro URL/타이틀/설명 반영
7. daily-macro 카드 1열 반영
8. daily-macro 카드에 배지 표시 반영
9. daily-macro 계산에 daily-meal 드롭다운 선택값 반영
10. 공통 하단 고정 CTA 스타일 정리

# 8. 최종 구현 기준
구현 결과는 아래 조건을 만족해야 한다.

- basic-info CTA 문구는 `하루 권장 칼로리 확인하기`여야 한다
- basic-info CTA 컬러는 daily-calories의 `하루 식단 구성하기` 버튼과 동일해야 한다
- basic-info 버튼은 하단 고정이어야 한다

- daily-calories의 `입력 다시하기` 버튼은 `정보 입력 다시하기`로 변경되어야 한다

- daily-meal에서는 식사 시간 입력이 없어야 한다
- daily-meal 각 식사 행에는 입력박스와 휴지통 버튼 사이에 드롭다운 2개가 있어야 한다
- 드롭다운 1 기본값은 `탄수화물 중심`이어야 한다
- 드롭다운 2 기본값은 `보통`이어야 한다
- 드롭다운 1 선택지는 `탄수화물 중심 / 단백질 중심 / 지방 중심`이어야 한다
- 드롭다운 2 선택지는 `적게 / 보통 / 많이`여야 한다
- daily-meal 식사 구조 카드 타이틀은 `하루 식사는 어떻게 하고 싶으세요?`여야 한다
- daily-meal CTA는 `하루 식단 확인하기 / 하루 권장 칼로리 확인하기`여야 한다
- daily-meal 버튼은 카드 밖 + 하단 고정이어야 한다
- daily-meal 버튼 간격은 daily-calories와 동일해야 한다

- daily-macro URL은 `daily-macro`여야 한다
- daily-macro 타이틀은 `Step4 한번에 얼마나 먹어야 하는지 확인하세요`여야 한다
- daily-macro 설명은 `평일과 주말 식사 구조에 맞춰 한 끼 기준을 계산했어요`여야 한다
- daily-macro 카드는 1열이어야 한다
- daily-macro 각 카드에는 식사 이름 / kcal / 탄수화물 / 단백질 / 지방이 있어야 한다
- kcal는 카드 안에서 가장 크게 강조되어야 한다
- 탄수화물 / 단백질 / 지방은 가로 3분할이어야 한다
- daily-macro 카드 우측 상단에는 선택값 2개가 배지 형태로 보여야 한다
- daily-meal의 `적게/보통/많이`는 0.8 / 1.0 / 1.2 가중치로 반영되어야 한다
- daily-meal의 `탄수화물 중심 / 단백질 중심 / 지방 중심`은 선택 영양소 +20% 강한 상향으로 반영되어야 한다
- daily-macro 버튼은 카드 밖 + 하단 고정이어야 한다
- daily-macro CTA는 위 `다음`, 아래 `뒤로가기`여야 한다

- basic-info / daily-meal / daily-macro 하단 CTA 영역은 공통 스타일이어야 한다
