# OnFit 디자인 시스템 v1

---

# 1️⃣ 브랜드 비주얼

## 1.1 브랜드 방향
- 화이트 기반 미니멀 UI
- 순간적인 재미 강조
- 장기 사용 가능한 안정적 구조

---

## 1.2 컬러 시스템

### Primary (블루)
- Blue-600: #2563EB
- Blue-500: #3B82F6
- Blue-100: #DBEAFE

### Accent (오렌지)
- Orange-500: #F59E0B

사용 조건:
- 기록 완료
- 목표 달성
- 캐릭터 변화
- 매칭 성공

※ 상시 사용 금지

### Neutral
- White: #FFFFFF
- Gray-100: #F1F5F9
- Gray-300: #CBD5E1
- Gray-600: #475569
- Gray-900: #0F172A

### 컬러 비율
- White/Gray: 80%
- Blue: 15%
- Orange: 5% 이하

---

## 1.3 타이포그래피

폰트: Pretendard

### 계층 구조
- Title: 20~24px / SemiBold
- Body: 14~16px / Regular
- Data: 16~18px / Medium
- Caption: 12~13px / Regular

---

# 2️⃣ UI 스타일 구조

## 2.1 구조 정의

모든 화면 공통 구조:

[상단] Header
[중앙] Content
[하단] CTA

---

## 2.2 레이아웃
- 좌우 여백: 16px
- 상하 간격: 24px
- 컴포넌트 간격: 16px
- 정렬: 좌측 정렬

---

## 2.3 컴포넌트

### 버튼
- Primary: Blue 배경 / White 텍스트 / 48px / radius 12px
- Secondary: Outline / Gray

### 카드
- 배경: Gray-100
- radius: 16px
- padding: 16px

### 리스트
- 텍스트 중심
- 높이: ~56px
- 여백으로 구분

---

## 2.4 정보 구조
- 카드 = 상태/요약
- 리스트 = 기록/데이터

---

## 2.5 CTA 규칙
- Primary: 항상 1개
- Secondary: 최대 2개

---

# 3️⃣ 캐릭터 시스템

## 3.1 정의
- 세미 리얼 3D
- 저디테일
- UI 컴포넌트 역할

---

## 3.2 스타일
- Soft 3D
- 매끈한 표면
- 저채도 색상

---

## 3.3 얼굴
- 성별 기반 고정
- 표정 변화 없음

---

## 3.4 변화 시스템

### 단계
- Level 1: 마름
- Level 2: 일반
- Level 3: 균형
- Level 4: 탄탄
- Level 5: 벌크

### 변화 요소
- 어깨
- 허리
- 팔/다리 두께

---

## 3.5 사용 위치
- 홈 상단
- 요약 화면
- 변화 확인 화면

---

# 4️⃣ 인터랙션 시스템

## 4.1 정의
- 필요한 순간만 반응

---

## 4.2 적용 시점
- 입력
- 완료
- 변화

---

## 4.3 규칙
- 속도: 100~300ms
- 동작: scale / 색 변화 / 미세 이동

---

## 4.4 금지
- 반복 애니메이션
- 과한 연출

---

## 4.5 컬러 인터랙션
- Blue: 기본 반응
- Orange: 성공/변화 순간
