# tdee_calculator daily-macro Page Spec v1

## 문서 목적
이 문서는 `tdee_calculator` 프로젝트의 **daily-macro 페이지 구현 지시용 MD 파일**이다.  
코덱스 CLI는 이 문서를 기준으로 daily-macro 페이지를 구현한다.

중요:
- 이 페이지는 daily-meal 페이지 다음 단계다.
- daily-meal에서 입력한 평일/주말 식사 구조를 기준으로 끼니별 섭취 기준을 계산하고 보여준다.
- 이 페이지의 목적은 사용자가 **한 끼에 얼마나 먹어야 하는지 확인**하게 만드는 것이다.

---

## 1. 페이지 역할
daily-macro 페이지의 역할은 아래와 같다.

1. 이전 페이지에서 설정한 목표를 달성하기 위해, 사용자가 한 끼에 얼마나 먹어야 하는지 보여준다
2. daily-meal에서 설정한 평일/주말 식사 구조를 기준으로 끼니별 기준을 계산해 보여준다
3. 사용자가 이 페이지를 보고  
   - `아, 한 끼에 이만큼 먹어야 하는구나`
   를 이해하게 만든다
4. 그 다음 페이지에서  
   - `그럼 어떤 걸 먹어야 하지?`
   로 자연스럽게 이어지게 한다

즉, 이 페이지는 **한 끼 기준 확인 페이지**다.

---

## 2. 페이지 성격
- 역할: 결과 확인형
- 정의: 식사 구조를 기준으로 끼니별 섭취 기준을 계산해서 보여주는 페이지
- 톤앤매너: 코치형
- 말투: 직설적이지만 과하지 않게
- 목적: 사용자가 한 끼 기준을 바로 이해하게 만드는 것

주의:
- 이 페이지에서 사용자가 비중(%), kcal, 탄단지를 수정하게 하지 않는다
- 확인에 집중하는 페이지로 유지한다

---

## 3. URL
- `daily-macro`

주의:
- 소문자 + 하이픈 기준 유지

---

## 4. 상단 구조

### 페이지 타이틀
`Step4 한번에 얼마나 먹어야 하는지 확인하세요`

### 페이지 설명
`평일과 주말 식사 구조에 맞춰 한 끼 기준을 계산했어요`

주의:
- 타이틀과 설명은 다른 주요 페이지들과 같은 방식으로 상단 헤더 영역에 배치한다
- 사용자가 왜 평일/주말로 나뉘어 있는지 설명에서 이해할 수 있어야 한다

---

## 5. 탭 구조

### 탭 구성
- `평일`
- `주말`

### 기본 선택 탭
- `평일`

### 데이터 기준
- 각 탭은 daily-meal에서 입력한 식사 구조를 그대로 가져온다
- 예:
  - 평일: 아침 / 점심 / 저녁
  - 주말: 아점 / 저녁

주의:
- daily-macro에서는 식사 이름을 다시 입력받지 않는다
- daily-meal에서 입력한 식사 이름을 그대로 사용한다

---

## 6. 계산 기준
### 계산 원칙
- 하루 총 권장 칼로리 기준을 식사 개수만큼 **균등 분배**
- 끼니별 kcal 기준에 맞춰 탄수화물 / 단백질 / 지방을 자동 비례 계산

### 예시
- 평일 3끼면 각 끼니를 1/3 기준으로 계산
- 주말 2끼면 각 끼니를 1/2 기준으로 계산

주의:
- 사용자가 비중(%)을 수정하는 구조는 넣지 않는다
- 자동 계산 결과를 그대로 보여주는 구조로 간다

---

## 7. 카드 구조
각 끼니는 **카드형**으로 표시한다.

### 카드 목적
- 한 끼 기준을 덩어리로 보여준다
- 사용자가 카드 하나만 봐도 “한 번에 얼마나 먹어야 하는지” 이해할 수 있어야 한다

주의:
- 리스트형 반복 구조로 가지 않는다
- 카드형으로 한 끼 기준을 더 명확하게 보여준다

---

## 8. 카드 안 정보 순서
카드 안 정보 순서는 아래와 같다.

1. 식사 이름
2. kcal
3. 탄수화물 / 단백질 / 지방

즉 구조는 아래 순서다.

- 식사 이름
- 큰 숫자의 kcal
- 탄수화물 / 단백질 / 지방

---

## 9. 카드 안 정보 표현 방식

### 9.1 식사 이름
예:
- `아침`
- `점심`
- `저녁`
- `아점`

### 9.2 kcal
- 카드 안에서 **가장 크게 강조**
- 사용자가 한 끼 총량을 가장 먼저 인식하게 한다

예:
- `600 kcal`

주의:
- 이 페이지의 핵심은 “한 번에 얼마나 먹어야 하는지”이므로 kcal가 가장 눈에 띄어야 한다

### 9.3 탄수화물 / 단백질 / 지방
- 카드 하단에 **가로 3분할**로 배치

형식 예:
- `탄수화물 60g`
- `단백질 40g`
- `지방 20g`

주의:
- `탄 / 단 / 지` 같은 축약형 대신 풀네임 기준으로 표시한다
- 세로 3줄 배치가 아니라 가로 3분할로 간다

---

## 10. 표시 정보
각 카드에는 아래 정보가 모두 보여야 한다.

1. 식사 이름
2. kcal
3. 탄수화물
4. 단백질
5. 지방

즉, kcal만 보여주는 것이 아니라  
**kcal + 탄수화물 + 단백질 + 지방을 모두 표시**해야 한다.

---

## 11. 카드 하단 보조 문구
- 없음

주의:
- 카드마다 별도의 설명 문구를 반복해서 넣지 않는다
- 이 페이지는 한 끼 기준 확인에 집중한다
- “무엇을 먹어야 하는지”는 다음 페이지에서 다룬다

---

## 12. CTA

### 버튼 문구
`다음`

### 추가 버튼
- `뒤로가기`

### 배치
- 기본 CTA `다음` 버튼 아래에 `뒤로가기` 버튼 추가

주의:
- 다음 페이지가 아직 미확정이므로, 현재는 중립적인 `다음`으로 유지한다
- `뒤로가기` 버튼은 사용자가 이전 단계(daily-meal)로 돌아가 수정할 수 있게 하기 위한 용도다

---

## 13. 디자인 방향
tdee_calculator 프로젝트의 현재 페이지 흐름과 OnFit 디자인 시스템 v1의 방향을 참고한다.

### 핵심 방향
- 화이트 기반 미니멀 UI
- 조용한 기본 위에 작은 포인트만 사용
- 카드형 구조 안에서 한 끼 기준이 명확하게 보여야 한다
- 과한 장식, 과한 애니메이션 금지

### 구현 포인트
- 평일/주말 탭 구조가 daily-meal 페이지와 자연스럽게 이어져야 한다
- kcal는 가장 크게, 탄수화물/단백질/지방은 그 아래 가로 3분할로 정돈되게 배치한다
- 카드 하단 반복 문구 없이 정보만으로 이해되게 한다

---

## 14. 금지 사항
아래 변경은 하지 않는다.

- daily-macro에서 식사 이름을 다시 입력받기
- 식사 시간 입력을 다시 추가하기
- 사용자가 비중(%)을 수정하게 만들기
- 사용자가 kcal를 직접 수정하게 만들기
- 사용자가 탄수화물/단백질/지방을 직접 수정하게 만들기
- 리스트형 반복 구조로 바꾸기
- `탄 / 단 / 지` 축약형만 사용하기
- 카드 하단에 보조 설명 문구를 반복해서 추가하기

---

## 15. 구현 우선순위
1. daily-macro URL 연결
2. 상단 타이틀 / 설명 반영
3. 평일 / 주말 탭 구조 반영
4. daily-meal 식사 구조 불러오기
5. 식사 개수 기준 균등 분배 계산
6. 카드형 UI 구현
7. kcal 강조 스타일 반영
8. 탄수화물 / 단백질 / 지방 가로 3분할 반영
9. `다음` + `뒤로가기` 버튼 연결

---

## 16. 최종 구현 기준
구현 결과는 아래 조건을 만족해야 한다.

- URL은 `daily-macro`여야 한다
- 상단 타이틀은 `Step4 한번에 얼마나 먹어야 하는지 확인하세요`여야 한다
- 상단 설명은 `평일과 주말 식사 구조에 맞춰 한 끼 기준을 계산했어요`여야 한다
- 탭은 `평일 / 주말` 2개여야 한다
- 기본 선택 탭은 `평일`이어야 한다
- daily-meal에서 입력한 식사 이름을 그대로 가져와야 한다
- 식사 개수 기준으로 균등 분배 계산을 해야 한다
- 각 끼니는 카드형으로 보여야 한다
- 카드 안 정보 순서는 `식사 이름 → kcal → 탄수화물 / 단백질 / 지방`이어야 한다
- kcal는 카드 안에서 가장 크게 강조되어야 한다
- 탄수화물 / 단백질 / 지방은 가로 3분할이어야 한다
- 탄단지는 `탄수화물 60g / 단백질 40g / 지방 20g` 형식의 풀네임 기준이어야 한다
- 카드 하단 보조 문구는 없어야 한다
- CTA는 `다음`이어야 한다
- `다음` 버튼 아래에 `뒤로가기` 버튼이 있어야 한다
