# tdee_calculator Step4 Meal Macro Page Spec v1

## 문서 목적
이 문서는 `tdee_calculator` 프로젝트의 **Step4 끼니별 섭취 기준 페이지 구현 지시용 MD 파일**이다.  
코덱스 CLI는 이 문서를 기준으로 Step4 페이지를 구현한다.

중요:
- 이 페이지는 daily-meal 페이지 다음 단계다.
- daily-meal에서 입력한 평일/주말 식사 구조를 기준으로 끼니별 섭취 기준을 계산하고 보여준다.
- 이 페이지는 단순 결과 확인 페이지가 아니라, **결과 + 조정 가능형 페이지**다.

---

## 1. 페이지 역할
이 페이지의 역할은 아래와 같다.

1. daily-meal에서 입력한 평일/주말 식사 목록을 불러온다
2. 식사 개수 기준으로 끼니별 섭취 기준을 **균등 분배**해서 보여준다
3. 사용자가 필요하면 **각 끼니의 칼로리 비중(%)을 조정**할 수 있게 한다
4. 비중 변경 시 kcal 및 탄/단/지 값을 자동으로 다시 계산한다

즉, 이 페이지는  
**자동 분배 결과 확인 + 필요 시 미세 조정**  
페이지다.

---

## 2. 상단 구조

### 페이지 타이틀
`Step4 끼니별 섭취 기준을 확인하고 조정해보세요`

### 페이지 설명
`입력한 식사 구조를 기준으로 끼니별 섭취 기준을 나눠봤어요`

주의:
- 타이틀과 설명은 다른 주요 페이지들과 같은 방식으로 상단 헤더 영역에 배치한다.
- 사용자가 이 페이지가 “자동 계산 결과를 보고 조정할 수 있는 페이지”라는 점을 바로 이해할 수 있어야 한다.

---

## 3. 탭 구조

### 탭 구성
- `평일`
- `주말`

### 기본 선택 탭
- `평일`

### 데이터 기준
- 각 탭은 daily-meal에서 입력한 식사 구조를 그대로 가져온다.
- 예:
  - 평일 탭: 아침 / 점심 / 저녁
  - 주말 탭: 아점 / 저녁

주의:
- 식사 이름과 식사 시간은 Step4에서 다시 입력받지 않는다.
- daily-meal에서 입력한 값을 그대로 사용한다.

---

## 4. 기본 분배 원칙

### 기본값
- **균등 분배**

예:
- 평일 3끼면 각 끼니 비중 기본값은 `33.3 / 33.3 / 33.4`
- 주말 2끼면 각 끼니 비중 기본값은 `50 / 50`

주의:
- 첫 진입 시 기본은 균등 분배로 자동 세팅한다.
- 사용자가 이후 직접 비중(%)을 조정할 수 있어야 한다.

---

## 5. 사용자 조정 방식

### 조정 가능한 값
- **각 끼니의 칼로리 비중(%)만 수정 가능**

### 조정 불가 값
- kcal
- 탄수화물
- 단백질
- 지방

위 값들은 모두 **자동 계산값**으로 처리한다.

즉:
- 사용자는 비중(%)만 수정
- 시스템은 비중을 기준으로 kcal와 탄/단/지를 자동 재계산

---

## 6. 자동 계산 규칙
각 끼니의 비중(%)이 정해지면 아래를 자동 계산한다.

- 끼니별 kcal
- 끼니별 탄수화물
- 끼니별 단백질
- 끼니별 지방

### 계산 원칙
- 하루 총 권장 칼로리 기준으로 끼니별 kcal를 계산
- 끼니별 kcal 비중에 맞춰 탄수화물 / 단백질 / 지방을 자동 비례 계산

주의:
- 사용자는 탄/단/지 값을 직접 수정하지 않는다.
- Step4는 비중 조정만 허용한다.

---

## 7. 행(row) 구조
각 끼니는 **리스트형 행(row)** 구조로 표시한다.

### 1줄
- 식사 이름
- 식사 시간

예:
- `아침 | 08:00`
- `점심 | 13:00`
- `저녁 | 19:00`

### 2줄
- 비중
- kcal
- 탄단지

즉 구조는 아래와 같다.

- 1줄: `식사 이름 / 시간`
- 2줄: `비중 / kcal / 탄단지`

주의:
- 카드형 반복 구조가 아니라, 리스트형 행(row) 구조를 사용한다.
- 식사 수가 많아져도 과하게 무거워 보이지 않게 해야 한다.

---

## 8. 2줄 정보 표현 방식
2줄 정보 영역은 **라벨 + 값 분리형**으로 표시한다.

### 예시
- `비중 33.3%`
- `600kcal`
- `탄수화물 60g / 단백질 40g / 지방 20g`

주의:
- `탄 / 단 / 지` 같은 축약형 대신 풀네임 기준으로 표시한다.
- 사용자가 바로 이해할 수 있는 표현을 우선한다.

---

## 9. 비중 입력 UI

### 방식
- 비중은 라벨 옆 작은 입력칸으로 표시한다.

### 예시
- `비중 [33.3] %`

### 의도
- 사용자가 “이 값은 조정 가능한 값”이라는 점을 바로 이해할 수 있어야 한다.
- 반대로 kcal / 탄단지는 입력창이 아니라 결과값으로 보여야 한다.

주의:
- 비중 입력칸은 너무 크지 않게, 한 행 안에서 보조 조정 UI처럼 보여야 한다.
- 하지만 터치/입력은 불편하지 않아야 한다.

---

## 10. 표시 정보
각 끼니에는 아래 정보가 모두 보여야 한다.

1. 식사 이름
2. 식사 시간
3. 비중(%)
4. kcal
5. 탄수화물
6. 단백질
7. 지방

즉, kcal만 보여주는 것이 아니라  
**kcal + 탄수화물 + 단백질 + 지방을 모두 표시**해야 한다.

---

## 11. 비중 합계 검증

### 입력 중
- 총합 100%가 아니어도 허용

### 다음 버튼 클릭 시
- 총합 100% 검증

즉,
- 수정 중에는 자유롭게 조정 가능
- 최종 이동 시점에만 검증

주의:
- 입력 중 실시간으로 강제 차단하지 않는다
- 다른 끼니 값을 자동 보정하지 않는다

---

## 12. 100% 검증 실패 시 경고 방식

### 방식
- 모달 경고

### 톤
- 코치형

### 모달 제목
`한번 더 확인해보세요`

### 모달 본문
`끼니별 칼로리 비중 합계가 100%가 되어야 해요`

### 모달 버튼
`수정하기`

주의:
- 이 경우는 다음 단계로 바로 진행할 수 없는 필수 검증이다
- 따라서 목표 설정 페이지처럼 “그대로 진행하기” 버튼은 사용하지 않는다

---

## 13. CTA

### 버튼 문구
`다음`

### 사용 이유
- 다음 페이지가 아직 미확정이므로, 현재는 중립적인 `다음`으로 유지한다.

주의:
- 이후 다음 단계가 확정되면 CTA 문구는 바뀔 수 있다.
- 현재 문서 기준으로는 `다음`을 사용한다.

---

## 14. 디자인 방향
tdee_calculator 프로젝트의 현재 페이지 흐름과 OnFit 디자인 시스템 v1의 방향을 참고한다.

### 핵심 방향
- 화이트 기반 미니멀 UI
- 조용한 기본 위에 작은 포인트만 사용
- 정보는 충분히 보여주되, 화면이 과하게 복잡해 보이지 않게 한다
- 리스트형 행 구조 안에서 정보 위계가 분명해야 한다

### 구현 포인트
- 1줄(식사 이름/시간)과 2줄(비중/kcal/탄단지)의 역할이 분명해야 한다
- 수정 가능한 값(비중)과 자동 계산값(kcal/탄단지)의 차이가 시각적으로 구분되어야 한다
- 평일/주말 탭 구조가 daily-meal 페이지와 자연스럽게 이어져야 한다

---

## 15. 금지 사항
아래 변경은 하지 않는다.

- Step4에서 식사 이름/시간을 다시 입력받기
- 비중 대신 kcal를 직접 수정하게 만들기
- 탄수화물/단백질/지방을 직접 수정하게 만들기
- 입력 중 실시간으로 총합 100%를 강제 차단하기
- 총합이 100%가 아닐 때 다른 끼니 값을 자동 비례 보정하기
- 카드형 반복 구조로 바꾸기
- 탄/단/지 축약형으로만 표시하기
- 100% 검증 실패 모달에 `그대로 진행하기` 버튼 추가하기

---

## 16. 구현 우선순위
1. Step4 상단 타이틀 / 설명 반영
2. 평일 / 주말 탭 구조 반영
3. daily-meal 식사 목록 불러오기
4. 기본 균등 분배값 자동 계산
5. 끼니별 리스트형 행 구조 구현
6. 비중 입력칸 구현
7. kcal / 탄수화물 / 단백질 / 지방 자동 계산 연결
8. 총합 100% 검증 로직 구현
9. 검증 실패 모달 구현
10. 페이지 하단 `다음` CTA 연결

---

## 17. 최종 구현 기준
구현 결과는 아래 조건을 만족해야 한다.

- 상단 타이틀은 `Step4 끼니별 섭취 기준을 확인하고 조정해보세요`여야 한다
- 상단 설명은 `입력한 식사 구조를 기준으로 끼니별 섭취 기준을 나눠봤어요`여야 한다
- 탭은 `평일 / 주말` 2개여야 한다
- 기본 선택 탭은 `평일`이어야 한다
- daily-meal에서 입력한 식사 이름/시간을 그대로 가져와야 한다
- 기본 비중은 식사 개수 기준 균등 분배여야 한다
- 사용자는 각 끼니의 비중(%)만 수정할 수 있어야 한다
- 비중 변경 시 kcal / 탄수화물 / 단백질 / 지방이 자동 변경되어야 한다
- 각 끼니는 리스트형 행(row) 구조여야 한다
- 행 구조는 1줄 `식사 이름 / 시간`, 2줄 `비중 / kcal / 탄단지`여야 한다
- 2줄 정보는 라벨 + 값 분리형이어야 한다
- 비중은 라벨 옆 작은 입력칸으로 보여야 한다
- kcal + 탄수화물 + 단백질 + 지방이 모두 표시되어야 한다
- 탄단지는 `탄수화물 60g / 단백질 40g / 지방 20g` 형식으로 표시되어야 한다
- 입력 중 총합 100%가 아니어도 허용해야 한다
- `다음` 클릭 시 총합 100% 검증을 해야 한다
- 총합이 100%가 아니면 모달을 띄워야 한다
- 모달 제목은 `한번 더 확인해보세요`여야 한다
- 모달 본문은 `끼니별 칼로리 비중 합계가 100%가 되어야 해요`여야 한다
- 모달 버튼은 `수정하기` 1개여야 한다
- 페이지 하단 CTA는 `다음`이어야 한다
