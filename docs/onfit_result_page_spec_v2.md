# OnFit Result Page Spec v2

## 문서 목적
이 문서는 `OnFit 식단 관리` 서비스의 **결과 페이지 구현 지시용 MD 파일**이다.  
코덱스 CLI는 이 문서를 기준으로 결과 페이지를 수정 및 구현한다.

---

## 1. 페이지 목적
결과 페이지의 목적은 아래 3가지다.

1. 사용자가 입력한 정보를 바탕으로 계산 결과를 명확하게 보여준다.
2. 다이어트 / 유지 / 벌크업 기준을 비교하고 이해하게 만든다.
3. 결과를 보고 끝나는 것이 아니라, 다음 행동으로 이어질 수 있게 만든다.

이 페이지는 **숫자만 보여주는 페이지가 아니라, 결과를 해석하고 선택하게 만드는 페이지**다.

---

## 2. 페이지 성격
- 역할: 결과 + 다음 행동 유도형
- 정의: 핵심 결과를 먼저 보여주고, 목표별 기준을 선택하게 한 뒤, 해당 기준을 해석해서 이해시키는 페이지
- 톤앤매너: 코치형
- 말투: 직설적이지만 과하지 않게
- 목적: 사용자가 숫자를 보고 바로 자신의 방향을 선택하게 만드는 것

---

## 3. 전체 구조
결과 페이지는 아래 순서로 구성한다.

1. 상단 숫자 요약 영역
2. 목표 선택 버튼 영역
3. 선택 결과 상세 영역
4. 입력 다시하기 버튼

즉, 구조는 아래 흐름으로 이해하면 된다.

`숫자 요약 먼저 → 목표 버튼 아래 → 버튼 클릭 시 상세 결과 노출`

주의:
- 기존에 있던 아래 문구 섹션은 삭제한다.
  - `덜 먹는 게 아니라, 내 기준보다 맞게 줄여야 해요`
  - `지금 몸 상태를 유지하고 싶다면 이 기준이 필요해요`
  - `무작정 많이 먹지 말고, 늘리는 기준부터 확인하세요`

---

## 4. 상단 숫자 요약 영역

### 구성
- `기초대사량`
- `유지 칼로리`

### 배치
- 같은 크기의 2개 카드로 나란히 표시한다.
- 두 카드는 동일한 비중으로 보이게 한다.
- 결과 페이지 상단에서 가장 먼저 보이도록 배치한다.

### 라벨 확정
- `기초대사량`
- `유지 칼로리`

주의:
- `BMR`, `TDEE` 영문 라벨 그대로 쓰지 않는다.
- `하루 섭취 칼로리`로 바꾸지 않는다.
- 상단 라벨은 사용자가 바로 이해할 수 있는 한글 기준으로 유지한다.

---

## 5. 목표 선택 버튼 영역

### 버튼 구조
- 가로 3분할 배치

### 버튼 라벨 확정
- `다이어트`
- `유지`
- `벌크업`

### 동작
- 사용자가 버튼 하나를 클릭하면, 해당 목표 기준의 상세 결과가 아래 영역에 노출된다.
- 다른 버튼을 클릭하면 아래 상세 결과 내용이 해당 목표 기준으로 바뀌어야 한다.
- 이 영역은 탭처럼 동작하지만, UI 표현은 버튼처럼 직관적으로 보여야 한다.

주의:
- `감량 / 유지 / 벌크` 라벨을 그대로 쓰지 않는다.
- 사용자 관점에서 더 익숙한 `다이어트 / 유지 / 벌크업`으로 통일한다.

---

## 6. 선택 결과 상세 영역 구조
버튼 클릭 후 하단 영역은 아래 순서로 구성한다.

### 상단
숫자 카드 영역

### 하단
설명 문구 영역

즉, 구조는 아래와 같다.

`숫자 카드 위 / 설명 문구 아래`

---

## 7. 숫자 카드 영역 정의

### 포함 정보
- 권장 칼로리 1개
- 탄수화물 g / %
- 단백질 g / %
- 지방 g / %

### 배치
- 권장 칼로리 카드는 상단에 크게 노출한다.
- 탄수화물 / 단백질 / 지방 카드는 그 아래에 배치한다.
- 매크로 카드는 반응형으로 처리한다.

### 반응형 기준
- 모바일: 탄 / 단 / 지 카드를 세로 1열로 배치
- 화면이 넓어지면: 탄 / 단 / 지 카드를 가로 3분할로 배치

### 매크로 카드 정보 순서
- `라벨 → g → %`

주의:
- 모바일에서 무조건 가로 3분할로 고정하지 않는다.
- 숫자 가독성이 우선이다.
- 매크로 카드에서 g와 % 모두 보여줘야 한다.

---

## 8. 설명 문구 영역 정의
숫자 카드 아래에는 목표별 설명 문구를 노출한다.

### 구조
- 설명 타이틀 1줄
- 설명 부가 1줄

### 다이어트
- 설명 타이틀: `다이어트를 위한 하루 권장 칼로리에요`
- 설명 부가: `처음부터 너무 많이 줄이면 오래 유지하기 어려워요`

### 유지
- 설명 타이틀: `현재 체중을 유지하기 위한 하루 권장 칼로리에요`
- 설명 부가: `활동량이 달라지면 이 기준도 달라질 수 있어요`

### 벌크업
- 설명 타이틀: `벌크업을 위한 하루 권장 칼로리에요`
- 설명 부가: `무작정 많이 먹기보다 천천히 늘리는 게 더 좋아요`

주의:
- 설명 타이틀은 숫자의 의미를 바로 이해시키는 역할이다.
- 설명 부가는 사용자가 숫자를 과하게 받아들이지 않게 조정하는 역할이다.

---

## 9. 하단 CTA
### 구성
- `입력 다시하기` 버튼 1개 유지

주의:
- 결과 페이지에서는 추가 CTA를 여러 개 늘리지 않는다.
- 현재 단계에서는 결과 확인에 집중한다.

---

## 10. 디자인 방향
온핏 디자인 시스템 v1의 방향을 참고한다.

### 핵심 방향
- 화이트 기반 미니멀 UI
- 조용한 기본 위에 작은 포인트만 사용
- Blue 계열 Primary / 선택 상태 강조
- Orange는 결과 페이지 기본 상태에서 적극 사용하지 않는다
- 과한 장식, 과한 애니메이션 금지

### 레이아웃 방향
- 상단 결과 요약과 하단 목표 선택이 명확히 구분되어야 한다
- 숫자는 한눈에 보이게 배치한다
- 선택 전 / 선택 후 상태 차이가 분명해야 한다
- 모바일에서 숫자 가독성이 깨지지 않아야 한다

### 컴포넌트 방향
- 숫자 카드는 카드형 구조로 보여준다
- 선택 버튼은 현재 선택 상태가 명확히 드러나야 한다
- 매크로 카드들은 같은 패턴으로 반복되어야 한다

---

## 11. 금지 사항
아래 변경은 하지 않는다.

- 결과 페이지를 단순 숫자 나열형으로 만들기
- 목표 버튼 없이 한 번에 모든 상세 결과를 길게 노출하기
- `감량 / 유지 / 벌크` 라벨 그대로 사용하기
- 상단에 영문 라벨 `BMR / TDEE` 그대로 사용하기
- 모바일에서도 매크로 카드를 무조건 가로 3분할 고정하기
- 결과 페이지에 과한 설명 문단 추가
- 여러 개의 다음 CTA 버튼 추가
- 기존 삭제 대상 문구 섹션을 다시 넣기

---

## 12. 구현 의도
이 페이지의 핵심은 **결과를 보여주는 것에서 끝나지 않고, 사용자가 자기 방향을 선택하게 만드는 것**이다.

즉,
- 숫자만 보여주고 끝내지 않는다
- 결과를 읽고 바로 선택하게 만든다
- 선택 후에는 해당 기준을 짧게 해석해준다

입력 페이지에서 계산 준비를 끝냈다면,  
결과 페이지에서는 결과 이해와 목표 선택을 한 번에 이어줘야 한다.

---

## 13. 구현 우선순위
결과 페이지 수정 시 우선순위는 아래 순서로 둔다.

1. 삭제 대상 문구 섹션 제거
2. 상단 결과 요약 구조 정리
3. `기초대사량 / 유지 칼로리` 라벨 반영
4. 목표 버튼을 `다이어트 / 유지 / 벌크업`으로 변경
5. 버튼 클릭 시 상세 영역 변경 로직 반영
6. 상세 영역 레이아웃을 `숫자 카드 위 / 설명 문구 아래`로 정리
7. 권장 칼로리 카드 + 매크로 카드 구성 반영
8. 매크로 카드 반응형 배치 반영
9. 설명 타이틀 / 설명 부가 문구 반영

---

## 14. 최종 구현 기준
구현 결과는 아래 조건을 만족해야 한다.

- 결과 페이지 상단에 `기초대사량 / 유지 칼로리` 2개 카드가 같은 크기로 나란히 보여야 한다
- 목표 선택 버튼은 `다이어트 / 유지 / 벌크업` 가로 3분할 구조여야 한다
- 버튼 클릭 시 아래 상세 결과가 해당 목표 기준으로 바뀌어야 한다
- 상세 영역은 `숫자 카드 위 / 설명 문구 아래` 구조여야 한다
- 숫자 카드에는 `권장 칼로리 1개 + 탄수화물 g/% + 단백질 g/% + 지방 g/%`가 포함되어야 한다
- 권장 칼로리 카드는 상단에 크게 보여야 한다
- 매크로 카드는 모바일에서는 세로 1열, 화면이 넓어지면 가로 3분할이어야 한다
- 매크로 카드 정보 순서는 `라벨 → g → %` 여야 한다
- 목표별 설명 타이틀 / 설명 부가 문구는 확정 문구 그대로 반영되어야 한다
- 삭제 대상 3개 문구 섹션은 결과 페이지에서 보이지 않아야 한다
