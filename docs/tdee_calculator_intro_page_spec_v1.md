# OnFit Intro Page Spec v1

## 문서 목적
이 문서는 `OnFit 식단 관리` 서비스의 **인트로 페이지 구현 지시용 MD 파일**이다.  
코덱스 CLI는 이 문서를 기준으로 인트로 페이지를 구현한다.

---

## 1. 페이지 목적
인트로 페이지의 목적은 아래 3가지다.

1. 서비스가 무엇인지 짧고 강하게 전달한다.
2. 사용자의 관심을 끄는 후킹 문구를 보여준다.
3. 사용자를 입력 페이지로 이동시킨다.

이 페이지는 **설명 페이지가 아니라 진입 페이지**다.  
따라서 정보량을 최소화하고, CTA 중심으로 구성한다.

---

## 2. 페이지 성격
- 톤앤매너: 코치형
- 말투: 직설적이지만 과하지 않게
- 인상: 미니멀 / 깔끔함 / 건강 서비스 느낌
- 목적: 사용자를 빠르게 다음 단계로 이동시키는 것

---

## 3. 페이지 구조
인트로 페이지는 **단일 화면 중심 구조**로 만든다.

### 레이아웃
- 화면 중앙 정렬
- 핵심 요소만 노출
- 스크롤이 길어지지 않도록 구성
- 첫 화면에서 타이틀, 설명, 버튼이 모두 보여야 한다

### 구성 요소
1. 서비스명
2. 타이틀
3. 설명
4. CTA 버튼 1개

---

## 4. 문구 확정본

### 서비스명
`OnFit 식단 관리`

### 타이틀
`언제까지 닭가슴살만 드실거에요?`

### 설명
`살 빼려면 어떤걸 얼마나 먹어야할지 알아보세요`

### CTA 버튼
`식단관리 시작하기`

---

## 5. 동작 정의
- CTA 버튼 클릭 시 입력 페이지로 이동한다.
- 입력 페이지 경로는 현재 프로젝트 라우팅 구조에 맞게 연결한다.
- 인트로 페이지에서는 추가 선택지 없이 CTA 버튼 1개만 제공한다.

---

## 6. 디자인 방향
온핏 디자인 시스템 v1의 방향을 참고한다.

### 핵심 방향
- 화이트 기반 미니멀 UI
- 조용한 기본 위에 작은 포인트만 사용
- CTA는 Blue 계열 중심
- Orange는 인트로 페이지에서 사용하지 않는다
- 과한 장식, 과한 애니메이션 금지

### 컬러 방향
- 배경: White
- 타이틀: Gray-900
- 설명: Gray-600
- 버튼: Blue Primary
- 보조 포인트: Blue-100 계열만 약하게 사용 가능

### 타이포 방향
- 서비스명: 작고 단정하게
- 타이틀: 가장 강조
- 설명: 타이틀보다 한 단계 낮은 크기
- 버튼: 명확하고 읽기 쉽게

---

## 7. 레이아웃 가이드
아래 원칙으로 구성한다.

### 상단
- 서비스명 `OnFit 식단 관리` 노출
- 메뉴는 두지 않거나 최소화한다
- 상단은 복잡하게 만들지 않는다

### 중앙
- 타이틀
- 설명
- 버튼
- 이 3개가 가장 중요하다

### 하단
- 현재는 별도 정보 영역을 만들지 않는다
- 필요 시 아주 약한 보조 요소만 허용한다

---

## 8. 허용되는 시각 요소
아래 정도만 허용한다.

- 아주 옅은 배경 포인트
- 작은 아이콘 또는 작은 그래픽 포인트
- 버튼 hover / press 시 미세 반응
- 미세한 scale 또는 색 변화

---

## 9. 금지 사항
인트로 페이지에서 아래 요소는 넣지 않는다.

- 긴 소개 문단
- 기능 카드 여러 개
- 감량 / 유지 / 벌크 상세 설명
- FAQ
- 후기
- 광고 배너
- 여러 개의 CTA 버튼
- 큰 일러스트
- 반복 애니메이션
- 과한 그래디언트
- 복잡한 헤더 메뉴

---

## 10. 구현 의도
이 페이지는 사용자를 오래 머무르게 하는 페이지가 아니다.  
핵심 문구로 관심을 끌고, 버튼 클릭으로 입력 페이지에 진입시키는 것이 목적이다.

즉,
- 설명보다 후킹이 먼저
- 정보량보다 전환이 먼저
- 브랜딩보다 행동 유도가 먼저

---

## 11. 구현 우선순위
인트로 페이지 구현 시 우선순위는 아래 순서로 둔다.

1. 문구 반영 정확도
2. 중앙 정렬 구조
3. CTA 버튼 가독성
4. 미니멀한 시각 완성도
5. 작은 보조 요소

---

## 12. 최종 구현 기준
구현 결과는 아래 조건을 만족해야 한다.

- 첫 화면에서 서비스명 / 타이틀 / 설명 / 버튼이 모두 보일 것
- CTA 버튼은 1개만 존재할 것
- 버튼 클릭 시 입력 페이지로 이동할 것
- 전체 인상은 깔끔하고 직관적일 것
- 문구의 후킹이 화면에서 충분히 살아날 것
