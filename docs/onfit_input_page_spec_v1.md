# OnFit Input Page Spec v1

## 문서 목적
이 문서는 `OnFit 식단 관리` 서비스의 **정보 입력 페이지 구현 지시용 MD 파일**이다.  
코덱스 CLI는 이 문서를 기준으로 정보 입력 페이지를 수정 및 구현한다.

---

## 1. 페이지 목적
정보 입력 페이지의 목적은 아래 3가지다.

1. 사용자가 계산에 필요한 정보를 빠르게 입력하게 한다.
2. 헷갈릴 수 있는 항목만 짧게 안내한다.
3. 입력 완료 후 결과 확인 단계로 자연스럽게 연결한다.

이 페이지는 **설명 페이지가 아니라 입력 실행 페이지**다.  
따라서 정보량은 최소화하고, 입력 흐름과 선택 경험을 우선한다.

---

## 2. 페이지 성격
- 역할: 혼합형
- 정의: 빠르게 입력하게 하되, 헷갈릴 수 있는 항목만 짧게 안내하는 페이지
- 톤앤매너: 코치형
- 말투: 직설적이지만 과하지 않게
- 목적: 사용자가 망설이지 않고 입력을 완료하게 만드는 것

---

## 3. 상단 문구 확정본
상단의 `INPUT` 라벨은 삭제한다.

### 타이틀
`닭가슴살부터 먹지 말고, 기준부터 확인하세요`

### 설명
`기본 정보와 활동량을 입력하면 하루 권장 칼로리를 계산할 수 있어요`

---

## 4. 전체 구조
정보 입력 페이지는 **2개 섹션 구조**로 구성한다.

### 섹션 1
`기본 정보`

### 섹션 2
`활동 및 추가 정보`

불필요하게 섹션을 많이 나누지 않는다.  
현재 목표는 카드 수를 줄여 심리적 입력 부담을 낮추는 것이다.

---

## 5. 기본 정보 섹션 정의

### 입력 순서
1. 성별
2. 나이
3. 키
4. 현재 체중

### 성별 입력 방식
- 가로 2분할 선택형으로 구현한다.
- 항목: `남성` / `여성`
- 동일한 너비로 배치한다.
- 선택 시 즉시 활성 상태가 드러나야 한다.
- 선택 전 상태와 선택 후 상태가 명확히 구분되어야 한다.

### 나이 입력 방식
- 직접 숫자 입력

### 키 입력 방식
- 직접 숫자 입력

### 현재 체중 입력 방식
- 직접 숫자 입력

### 배치
- `나이`는 한 줄 단독 배치
- `키`와 `현재 체중`은 한 줄 2열 배치

---

## 6. 활동 및 추가 정보 섹션 정의

### 항목 1
`평소 활동량`

### 입력 방식
- 현재처럼 세로 카드형 리스트를 유지한다.
- 드롭다운으로 바꾸지 않는다.
- 가로 탭형으로 바꾸지 않는다.
- 사용자는 각 선택지를 보고 바로 자신의 생활 패턴과 비교할 수 있어야 한다.

### 선택지 개수
- 5단계 유지

### 선택지 라벨 확정본
아래 문장을 그대로 사용한다.

1. `하루 대부분 앉아서 보내요`
2. `앉아 있는 시간이 많지만 조금씩 움직여요`
3. `앉아 있는 시간과 움직이는 시간이 비슷해요`
4. `움직이는 시간이 많은 편이에요`
5. `하루 대부분 몸을 많이 쓰는 편이에요`

주의:
- 라벨은 추상어(낮음/보통/높음)로 바꾸지 않는다.
- 라벨 자체가 생활 문장 역할을 해야 한다.
- 운동 횟수 중심 설명으로 바꾸지 않는다.
- 현재 단계에서는 운동량 세부 확인보다 생활 패턴 기준 선택이 목적이다.

---

## 7. 체지방률 입력 정의

### 노출 방식
- `활동 및 추가 정보` 섹션 안에 기본 노출한다.
- 숨김 처리하거나 토글로 감추지 않는다.

### 라벨
`체지방률 (선택)`

### 보조 문구
`모르면 비워두셔도 괜찮아요`

### 입력 방식
- 직접 숫자 입력

주의:
- 선택 입력임을 명확히 보여줘야 한다.
- 필수 입력처럼 느껴지지 않게 구성한다.

---

## 8. CTA 버튼 정의

### 버튼 문구
`내 기준 보러가기`

### 버튼 위치
- 페이지 하단 고정 CTA 또는 하단 명확한 위치에 배치
- 사용자가 입력 완료 후 바로 누를 수 있어야 한다

### CTA 위 보조 문구
- 넣지 않는다

---

## 9. 디자인 방향
온핏 디자인 시스템 v1의 방향을 참고한다.

### 핵심 방향
- 화이트 기반 미니멀 UI
- 조용한 기본 위에 작은 포인트만 사용
- Blue 계열 Primary CTA 사용
- Orange는 입력 페이지의 기본 상태에서 적극 사용하지 않는다
- 과한 장식, 과한 애니메이션 금지

### 레이아웃 방향
- 좌우 여백은 충분히 유지
- 섹션 간 간격은 명확하게 구분
- 카드 수를 줄여 한 화면이 과하게 길어 보이지 않게 만든다
- 입력과 선택이 빠르게 이뤄지는 흐름을 우선한다

### 컴포넌트 방향
- 선택형 UI는 상태 변화가 명확해야 한다
- 숫자 입력창은 읽기 쉽고 깔끔해야 한다
- 카드형 활동량 선택 UI는 선택 여부가 분명히 보여야 한다

---

## 10. 금지 사항
아래 변경은 하지 않는다.

- 상단에 `INPUT` 라벨 유지
- 섹션을 다시 4개로 늘리기
- 활동량 선택지를 추상어만으로 표기하기
- 활동량 설명을 운동 횟수 중심으로 바꾸기
- 체지방률을 숨김 토글로 바꾸기
- CTA 위 보조 문구 추가
- 입력 페이지에 불필요한 긴 설명 추가
- 입력 페이지에 광고성 요소 추가
- 여러 개의 CTA 버튼 추가

---

## 11. 구현 의도
이 페이지의 핵심은 **사용자가 망설이지 않고 입력을 끝내게 만드는 것**이다.

즉,
- 설명보다 입력 흐름이 먼저
- 구조보다 선택 경험이 먼저
- 정보량보다 완료율이 먼저

인트로 페이지에서 후킹했다면,  
입력 페이지에서는 그 흐름을 이어 받아 바로 행동으로 연결해야 한다.

---

## 12. 구현 우선순위
정보 입력 페이지 수정 시 우선순위는 아래 순서로 둔다.

1. 상단 문구 변경
2. `INPUT` 라벨 삭제
3. 2개 섹션 구조 반영
4. 성별 입력 UI를 가로 2분할로 변경
5. 기본 정보 입력 순서 정리
6. 키 / 현재 체중 2열 배치 반영
7. 평소 활동량 라벨 5개 문구 변경
8. 체지방률 라벨 및 보조 문구 반영
9. CTA 버튼 문구 변경

---

## 13. 최종 구현 기준
구현 결과는 아래 조건을 만족해야 한다.

- 상단 `INPUT` 라벨이 없어야 한다
- 타이틀과 설명이 확정 문구로 교체되어야 한다
- 섹션 구조가 `기본 정보 / 활동 및 추가 정보` 2개로 정리되어야 한다
- 성별은 가로 2분할 선택형이어야 한다
- 기본 정보 입력 순서가 `성별 → 나이 → 키 → 현재 체중` 이어야 한다
- `키 / 현재 체중`은 한 줄 2열이어야 한다
- `평소 활동량`은 세로 카드형 리스트를 유지해야 한다
- 활동량 라벨 5개는 확정 문구 그대로 반영되어야 한다
- 체지방률은 기본 노출되어야 하며, `체지방률 (선택)` 라벨과 `모르면 비워두셔도 괜찮아요` 문구가 보여야 한다
- 하단 CTA 버튼 문구는 `내 기준 보러가기` 여야 한다
